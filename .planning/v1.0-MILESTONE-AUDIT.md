## Integration Breaks (6)

### BREAK 1: IPC Argument Name Mismatches (2 commands)

| Command | Frontend Sends | Tauri Maps To | Rust Expects | Status |
|---------|---------------|---------------|-------------|--------|
| toggle_favorite | `{docId}` | `doc_id` | `id` | BROKEN |
| record_search_click | `{documentId}` | `document_id` | `doc_id` | BROKEN |

**Fix:** 1-line changes in `client/hooks/useTauri.ts`
**Affects:** INTL-02, PAGE-06, PAGE-11

### BREAK 2: TopBar Indexing Indicator Never Activates

`useIndexingStore.setProgress()` is never called. No component registers `listen("index-progress", ...)` to bridge Tauri events to the Zustand store.

**Fix:** Add `useEffect` in AppShell with `listen("index-progress", ...)` updating the store
**Affects:** UX-04, FWAT-05

### BREAK 3: IndexProgress Event Field Name Mismatch

`IndexProgress` struct missing `#[serde(rename_all = "camelCase")]`. Fields serialize as `folder_id` but WatchedPage reads `folderId`. Also: WatchedPage checks `status === "complete"` but backend emits `"scan-complete"`.

**Fix:** Add serde annotation + fix status string comparison
**Affects:** FWAT-05, PAGE-08

### BREAK 4: path_index Not Rebuilt on Startup

`DocumentIndexer.rebuild_path_index()` exists and is tested but never called in `lib.rs` setup hook. On app restart, all files are re-embedded unnecessarily.

**Fix:** 1-line call in `lib.rs` setup hook
**Affects:** FWAT-06

### BREAK 5: Settings Not Persisted

`update_settings` accepts a `Settings` struct but immediately discards it. `get_settings` returns hardcoded defaults. No disk persistence.

**Fix:** Implement JSON file read/write in app_data_dir
**Affects:** PAGE-10, DPIP-07

### BREAK 6: Onboarding Inside AppShell Layout

`/onboarding` route is a child of `<Route element={<AppShell />}>`, rendering with Sidebar + TopBar visible instead of fullscreen wizard.

**Fix:** Move `/onboarding` route outside the AppShell wrapper
**Affects:** PAGE-12
