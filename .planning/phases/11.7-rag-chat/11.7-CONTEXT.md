# Phase 11.7: Chat with Your Docs (RAG) - Context

**Gathered:** 2026-07-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Phase 11.7 adds a ChatGPT-style chat interface where users ask natural-language questions about their indexed corpus. Cortex retrieves relevant chunks from top-K docs, sends them to the active LLM provider with the question, streams the response with clickable citations back to source docs.

### What Phase 11.7 delivers

1. **`ChatEngine`** — Rust module `chat/engine.rs`. Per-query flow:
   - Embed question via MiniLM (existing EmbeddingService)
   - HNSW top-8 docs
   - Chunk each doc's text into 500-char windows, embed windows, pick top-3 per doc by cosine
   - Assemble RAG prompt w/ chunks + citations
   - Stream response via `ai_request_stream()` (new streaming variant of Phase 7 ai_request)
2. **Streaming Tauri event** — `chat-stream-token` emitted per token; `chat-stream-complete` on finish w/ citation metadata
3. **`ChatSessionStore`** — JSON sidecar `app_data_dir/chat_sessions.json`. Each session = `{id, title, created_at, messages: [{role, content, citations?}]}`
4. **4 IPC commands** — `start_chat(query, session_id?)`, `list_chat_sessions()`, `delete_chat_session(id)`, `rename_chat_session(id, title)`
5. **`/chat` route + ChatPage** — message list top, input at bottom, streaming markdown w/ code highlighting (react-markdown + rehype-highlight; both may already be in deps for FilePreview)
6. **Citation UX** — inline `[1] [2]` chips clickable → nav to source doc + highlight cited chunk via query param
7. **Sidebar chat sessions drawer** — collapsible list below Saved Searches
8. **New-chat + reset** — top-right of ChatPage

### Out of scope
- Multi-modal (image queries) — deferred to v1.3 w/ rupixel
- Function calling / tool use inside chat (v1.3)
- Multi-turn context management beyond current session (v1.3 — long chats truncate)
- User-editable prompts (v1.3)
- Chat export (v1.3)
- Cross-session memory / user profiles (v2)

</domain>

<decisions>
## Implementation Decisions

### RAG Retrieval (D-01..D-05)

- **D-01: Top-K = 8 docs, top-3 chunks per doc = 24 chunk candidates.** Reranked by chunk-level cosine, keep top-12 for prompt (context budget ~6K tokens).
- **D-02: Chunk size = 500 chars w/ 50-char overlap.** Small enough for precise citation, large enough for context. Chunks computed on demand at query time (not persisted — cheap to recompute).
- **D-03: Cosine floor 0.35** — if best chunk is below this, respond "I couldn't find anything relevant in your library" without calling LLM.
- **D-04: Metadata filter passthrough** — chat inherits current search filters (topic, entity, date range) if the user is chatting from within a filtered view. Otherwise unscoped.
- **D-05: No re-ranking model beyond cosine.** LLM does final semantic understanding via the RAG prompt.

### RAG Prompt (D-06..D-08)

- **D-06: System prompt (inline const `RAG_SYSTEM_PROMPT`):**
  ```
  You answer questions using ONLY the provided document excerpts.
  Cite sources inline as [1], [2], [3] matching the numbered docs below.
  If the answer isn't in the documents, say "I don't have that in your library."
  Never fabricate facts. Never cite a source not in the list.
  Prefer specific quotes over paraphrases when quoting.
  ```
- **D-07: User message structure:**
  ```
  Documents in your library:
  [1] {title_1}: {chunk_1_text}
  [2] {title_2}: {chunk_2_text}
  ...
  [N] ...

  Question: {user_query}
  ```
- **D-08: Streaming.** Use SSE-style token stream from Anthropic/OpenAI/Gemini. Falls back to non-streaming for Ollama (or streams if supported).

### Streaming Infrastructure (D-09..D-11)

- **D-09: Extend `ai/service.rs` with `ai_request_stream(req) -> impl Stream<Item = Result<StreamChunk, Error>>`.** Per-provider implementation (`anthropic_chat_stream`, `openai_chat_stream`, etc.) using their SSE endpoints.
- **D-10: Tauri event schema.**
  - `chat-stream-token`: `{ session_id, message_id, token, cumulative_index }`
  - `chat-stream-complete`: `{ session_id, message_id, citations: [{ index, doc_id, chunk_start, chunk_end }], usage: { input_tokens, output_tokens } }`
  - `chat-stream-error`: `{ session_id, message_id, error }`
- **D-11: Frontend event listener via `useEffect` in ChatPage.** Appends tokens to the current-message state as they arrive. React 19 concurrent rendering absorbs the update cadence smoothly.

### Session Store (D-12..D-14)

- **D-12: Sidecar `app_data_dir/chat_sessions.json`.** Same load/save pattern as SavedSearchStore. Load once at boot into AppState.
- **D-13: Session title auto-generated from first user question** (first 40 chars, trimmed at word boundary). User can rename via inline edit.
- **D-14: No session eviction.** Users can manually delete via trash-icon on session row. All sessions persist across restarts.

### UI (D-15..D-18)

- **D-15: Route `/chat` and `/chat/:sessionId`.** Root `/chat` opens fresh session; `:sessionId` loads existing.
- **D-16: Message list** — user messages right-aligned indigo; assistant left-aligned neutral. Streaming assistant messages show cursor blink at end while receiving.
- **D-17: Citations render as inline chips** — small pill w/ number + hover tooltip showing doc title. Click → `/document/:id?highlight={chunk_start}-{chunk_end}` (DocumentPage FilePreview reads highlight param to scroll to that chunk).
- **D-18: Sidebar entry** — "Chat" link w/ chat-bubble icon between Saved Searches and Ontology (Phase 11.6). Collapsible drawer shows recent 5 sessions inline; overflow to full list on click.

<specifics>
## Specific Ideas

- **RAG killer feature: cited answers.** Users get quotes with clickable sources, not vague summaries. Zero-fabrication policy in prompt enforces this.
- **"I don't have that in your library" is a first-class answer.** Trust matters more than coverage.
- **Chat inherits current filters** — powerful for e.g. "search Property topic → chat 'when did I buy Unit 204?'".
- **Streaming feels magical** — token-by-token Anthropic response is standard now; users expect it.
- **Sessions persist by default.** Users can revisit last week's chat about their taxes without losing it.
- **New-chat button in top-right of ChatPage** — standard ChatGPT UX; low friction.
- **Cost transparency** — each response can show input/output token count in a subtle footer (opt-in).

</specifics>
