# Phase 11.7 Plan 03: DocumentPage Citation Highlight Summary

**DocumentPage parses `?highlight={start}-{end}` from the URL via useSearchParams and forwards it to a new FilePreview.highlightRange prop; TextPreview marks the exact character range with a yellow highlight + 2s pulse and scrolls it into view, MarkdownPreview marks and scrolls the whole rendered block (splitting markdown source would corrupt syntax), and PDF/image/unsupported previews ignore it entirely.**

## What Was Built

### Task 1 — FilePreview highlightRange plumbing
- `FilePreview` dispatcher gained an optional `highlightRange?: { start: number; end: number }` prop (exported as `HighlightRange` type for reuse by children).
- Forwarded only to `TextPreview` (docType `txt`/`csv`) and `MarkdownPreview` (docType `md`). `PdfPreview`, `ImagePreview`, and `UnsupportedPreview` signatures are untouched — highlightRange is a v1.3 problem for PDFs per the plan.
- **TextPreview:** validates the range (`Number.isFinite`, `start >= 0`, `start < end`, `end <= text.length`) then splits the raw `data.text` into three pieces rendered as `{before}<span class="chat-citation-highlight bg-yellow-200 ...">{highlighted}</span>{after}` inside the existing `<pre>`. A `useEffect` scrolls the middle span into view (`block: "center"`) once mounted.
- **MarkdownPreview:** same validation, but because `ReactMarkdown` parses the string into an AST, splitting mid-string would break formatting (e.g. slicing inside `**bold**`). Instead the whole `.prose` container gets the `chat-citation-highlight` class + a translucent yellow background + pulse animation, and that container is scrolled into view. This satisfies D-17's Claude's Discretion note explicitly allowing a "scroll-to" fallback.
- Added a `citation-pulse` keyframe animation to `client/global.css`'s `@theme` block (mirrors the existing `accordion-down`/`accordion-up` pattern) — a 2s box-shadow pulse that fades from a visible yellow ring to transparent.

### Task 2 — DocumentPage query param wiring
- `useSearchParams` (react-router-dom) + `useMemo` parse `?highlight=100-250` via `^(\d+)-(\d+)$` regex, guarded by `Number.isFinite` and `end > start`. Any failure returns `undefined`.
- Forwarded to `<FilePreview doc={doc} highlightRange={highlightRange} />`. No other DocumentPage behavior touched (metadata sidebar, breadcrumb, Open/Reveal buttons all unchanged).

## Deviations from Plan

### Auto-fixed Issues

**1. [Adaptation, not a bug] FilePreview prop signature mismatch**
- **Found during:** Task 1, before writing any code
- **Issue:** PLAN.md's `<interfaces>` section showed `<FilePreview docType={doc.docType} path={doc.path} name={doc.name} />` as the "existing usage" to preserve, but the actual `FilePreview` component takes a single `doc: Document` prop. The task prompt's file list also referenced `DocumentPreview.tsx`, which doesn't exist — the real component is `FilePreview.tsx`.
- **Fix:** Read the actual files first (as the plan's own guardrail instructs) and added `highlightRange` as a new optional prop on the real `doc`-based signature, leaving the existing `doc` prop and all call sites untouched.
- **Files modified:** client/components/preview/FilePreview.tsx, client/pages/DocumentPage.tsx
- **Commits:** 8356c3f, c9e7a39

**2. [Rule 1 - Bug] scrollIntoView crashes in environments that don't implement it**
- **Found during:** Writing automated test coverage for the checkpoint (see below)
- **Issue:** `Element.scrollIntoView` is not implemented in jsdom, and calling it unconditionally threw `TypeError: el?.scrollIntoView is not a function`, crashing the whole preview render inside a `useEffect` (which React surfaces as an error boundary trip in production).
- **Fix:** Guarded both TextPreview's and MarkdownPreview's scroll effects behind `typeof el.scrollIntoView === "function"` before calling it. Missing implementation now degrades gracefully to "highlight shown, no auto-scroll" instead of crashing.
- **Files modified:** client/components/preview/TextPreview.tsx, client/components/preview/MarkdownPreview.tsx
- **Commit:** 89de7fb
