Create `src-tauri/src/chat/session_store.rs` following the SavedSearchStore template. Implementation notes:

**Struct declaration:**
```rust
#[derive(Debug, Clone, Serialize, Deserialize, Default)]
#[serde(rename_all = "camelCase")]
pub struct ChatSessionStore {
    /// Ordered list of chat sessions. Most-recently-updated first is NOT enforced
    /// here — the store preserves insertion order. Sort/filter in the IPC layer.
    pub sessions: Vec<ChatSession>,
}
```

**File path:** `app_data_dir.join("chat_sessions.json")`.

**Methods to implement (mirror SavedSearchStore signatures):**

- `pub fn load(app_data_dir: &Path) -> Self` — read_to_string → from_str → unwrap_or_default. Never panic on I/O or JSON parse errors.
- `pub fn save(&self, app_data_dir: &Path) -> std::io::Result<()>` — to_string_pretty → write.
- `pub fn get(&self, id: &str) -> Option<&ChatSession>` — linear scan.
- `pub fn all(&self) -> &[ChatSession]` — return `&self.sessions`.
- `pub fn remove(&mut self, id: &str) -> bool` — retain + compare before/after len.
- `pub fn insert(&mut self, session: ChatSession)` — upsert by id (find position → replace; else push).

**New methods (chat-specific, no equivalent in SavedSearchStore):**

- `pub fn create_session(&mut self, id: String, title: String, now_iso: String) -> ChatSession` — builds a `ChatSession` with `id`, `title`, `created_at = now_iso.clone()`, `updated_at = now_iso`, `messages: vec![]`. Inserts via upsert. Returns a clone. The caller is responsible for generating `id` (e.g., `format!("cs-{}", Uuid::new_v4())`) and `now_iso` (e.g., `chrono::Utc::now().to_rfc3339()`) so the store stays pure and testable.

- `pub fn append_message(&mut self, session_id: &str, msg: ChatMessage, now_iso: String) -> bool` — find session by id; if found, push msg into session.messages and set session.updated_at = now_iso; return true. If not found, return false.

- `pub fn rename_session(&mut self, id: &str, new_title: String) -> Result<bool, String>` — trim whitespace; if resulting title is empty, return `Err("title must not be empty".to_string())`; else find session and update title; return `Ok(true)` if updated, `Ok(false)` if not found.

**Imports:**
```rust
use crate::types::{ChatSession, ChatMessage};
use serde::{Deserialize, Serialize};
use std::path::Path;
```

**Test conventions:**
- Use `tempfile::TempDir` (already in dev-dependencies) for tests.
- Helper `fn make_session(id: &str, title: &str) -> ChatSession` and `fn make_message(id: &str, role: ChatRole, content: &str) -> ChatMessage` to keep test bodies short.
- Every test names one behavior (see `<behavior>` above).
- Assert full field-by-field equality after round-trip (mirror SavedSearchStore test 3).

Do NOT wire this into AppState or lib.rs. Do NOT add IPC commands. Do NOT reference tokio locks — the outer `Arc<tokio::sync::Mutex<>>` wrapping is Plan 05's responsibility (mirrors saved_search_store pattern).
