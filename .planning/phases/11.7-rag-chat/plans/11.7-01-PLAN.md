Types to add (name, purpose, fields):

- `ChatRole` (enum, `#[serde(rename_all = "lowercase")]`): variants `User`, `Assistant`. Used to type ChatMessage.role safely on both sides.

- `Citation` (struct): fields `index: u32` (1-based, matches "[1]" in the streamed text), `doc_id: String`, `doc_title: String`, `chunk_start: u32`, `chunk_end: u32` (character offsets into the source doc, used by DocumentPage highlight — per D-17).

- `ChatMessage` (struct): fields `id: String` (mid-{uuid}), `role: ChatRole`, `content: String`, `citations: Option<Vec<Citation>>` (None for user messages, Some for assistant), `created_at: String` (RFC3339).

- `ChatSession` (struct): fields `id: String` (cs-{uuid}), `title: String` (auto-generated from first user message per D-13), `created_at: String`, `updated_at: String`, `messages: Vec<ChatMessage>`.

- `ChatStreamTokenPayload` (struct — event payload for `chat-stream-token` per D-10): fields `session_id: String`, `message_id: String`, `token: String`, `cumulative_index: u32`.

- `ChatStreamCompletePayload` (struct — event payload for `chat-stream-complete` per D-10): fields `session_id: String`, `message_id: String`, `citations: Vec<Citation>`, `input_tokens: Option<u64>`, `output_tokens: Option<u64>`.

- `ChatStreamErrorPayload` (struct — event payload for `chat-stream-error` per D-10): fields `session_id: String`, `message_id: String`, `error: String`.

- `StartChatArgs` (struct — IPC request shape for `start_chat`): fields `query: String`, `session_id: Option<String>` (None = new session per D-15), `filters: Option<SearchFilters>` (D-04 filter passthrough — reuse existing SearchFilters struct).

Do NOT wire these into `commands/mod.rs` or `lib.rs` — that is Plan 05's job. Only add the type definitions.
