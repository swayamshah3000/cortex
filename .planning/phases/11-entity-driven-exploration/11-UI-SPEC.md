## Spacing Scale

Cortex uses a 4px base grid. Inherited from Phase 8/9 contracts. **All phase 11 spacing values are strict multiples of 4. No sub-grid values are declared.**

| Token | Value | Usage in Phase 11 |
|-------|-------|------------------|
| xs | 4px | Icon-to-text gap in entity filter pill (`gap-1`), pill vertical padding (`py-1`), score badge inner padding |
| sm | 8px | Filter pill horizontal padding (`px-2`), save-search Dialog input spacing, section header bottom gap |
| md | 12px | Related doc row horizontal padding (`px-3`), saved search row padding (`px-3 py-2`) |
| lg | 16px | EntityDetailPage header gap, Dialog content padding, page section gap |
| xl | 24px | EntityDetailPage section separation, card-level padding (`p-6`) |
| 2xl | 32px | Page-level section breaks (`space-y-8`) |

**Exceptions:**
- Touch targets for filter pill remove-button (X icon): minimum 44×44px wrapper — 44 = 4 × 11, grid-aligned.
- Sidebar saved search rows: `py-2` (8px vertical, sm token). Same rhythm as existing Smart Spaces rows in Sidebar.

**No sub-grid values.** Phase 11 does NOT use `py-0.5` (2px), `px-2.5` (10px), or `gap-2` (6px). The existing `ScoreBadge` in `SearchPage.tsx` uses `py-0.5` — that component is not modified by phase 11; only new components added in this phase must be grid-aligned.

## Visual Hierarchy

**SearchPage — active entity filter pills atop results:** The active-filter pill bar is the highest-weight new element on the search page. Pills use `bg-accent-subtle text-accent-primary border-accent-primary/20` — matching topic chip styling (Phase 8 contract) to signal "active filter". The remove button `X` on each pill is a secondary action (neutral color, visible on pill hover).

**SearchPage — "Save this search" button:** Rendered in the page header row alongside the query description. Use `btn-secondary` variant (not `btn-primary`) at rest to avoid competing with the primary search interaction. Accent fill only appears inside the Dialog on the "Save search" button.

**DocumentPage Related panel:** Each row is a `<Link>` with `hover:bg-bg-tertiary` — same rhythm as existing Spaces link rows in the sidebar. The score badge is right-aligned within the row. The snippet (2-line clamp) has lower visual weight than the title.

**EntityDetailPage hierarchy:**
1. Entity class icon (24px, entity semantic color) + entity value (`text-2xl font-semibold text-text-primary`) — primary heading
2. Alias list (if any) — secondary, `text-sm text-text-secondary`
3. "Documents" section header — tertiary structural element
4. Co-occurring entities panel — lowest weight, rendered below the document list

**Sidebar "Saved Searches" section:** visually mirrors the existing "Spaces" section in `Sidebar.tsx`. Section header: `text-xs font-semibold uppercase tracking-wider text-text-tertiary`. Each row: bookmark icon (14px) + name + count `(N)`. Active row: `bg-bg-tertiary text-text-primary`. At-rest row: `text-text-secondary hover:bg-bg-tertiary`.

## Control Specifications

### 1. EntityChip — Dual Navigation (ENEX-01, D-01..D-03, D-17)

**Current behavior:** `EntityChip` renders as a `<Link>` to `/entities/{canonicalId}` (Phase 6 knowledge graph entity page).

**Phase 11 change:** Replace single-click navigation with dual-mode interaction:

| Interaction | Action | Target |
|-------------|--------|--------|
| Left click | Navigate to `/search?entity={class}:{value}` | Filter search results by this entity |
| Right click / `onContextMenu` | Navigate to `/entity/{class}/{value}` (URL-encoded) | Entity detail page (new phase 11 route) |

**Implementation:** Change the outer element from `<Link>` to `<button>` to intercept click. Use `useNavigate()` for both navigations.

```tsx
<button
  onClick={() => navigate(`/search?entity=${encodeURIComponent(`${resolvedClass}:${entity.value}`)}`)}
  onContextMenu={(e) => {
    e.preventDefault();
    navigate(`/entity/${encodeURIComponent(resolvedClass)}/${encodeURIComponent(entity.value)}`);
  }}
  aria-label={`Filter by ${resolvedClass}: ${entity.value}. Right-click for entity detail page.`}
  className="inline-flex items-center gap-2 px-2 py-1 rounded-full border border-border-secondary bg-bg-tertiary hover:bg-accent-subtle focus-visible:ring-2 focus-visible:ring-accent-primary focus-visible:ring-offset-2 focus-visible:outline-none transition-colors cursor-pointer"
>
  {icon}
  <span className="text-sm text-text-primary truncate max-w-[160px]">{entity.value}</span>
  {showSubclassBadge && (
    <Badge variant="outline" className="text-xs font-mono ml-1">
      {entity.subclass}
    </Badge>
  )}
</button>
```

**Padding correction:** Existing EntityChip uses `px-2.5` (10px — sub-grid). Phase 11 corrects this to `px-2` (8px, sm token) on the new button element. The sub-grid value came from the existing `<Link>` and was a pre-existing deviation; the visual change is imperceptible (2px narrower) and aligns the component to the 4px grid.

**Visual state change (active filter):** When an EntityChip's `{class}:{value}` matches an active `?entity=` URL param on SearchPage, the chip renders with `bg-accent-subtle text-accent-primary border-accent-primary/20` instead of the default `bg-bg-tertiary border-border-secondary`. Implement by passing an `isActive?: boolean` prop to `EntityChip`.

### 2. SearchPage — Active Entity Filter Bar (ENEX-01, D-04)

**Location:** Between `TopicFilterBar` and the result count line. Renders only when `searchParams.getAll('entity').length > 0`.

**`EntityFilterPill` spec:**

```tsx
<div className="inline-flex items-center gap-1 px-2 py-1 rounded-full border border-accent-primary/20 bg-accent-subtle text-accent-primary text-xs">
  {classIcon}  {/* entity class icon at 10px — same icon map as EntityHintChip in Phase 9 */}
  <span className="truncate max-w-[140px]">{class}: {value}</span>
  <button
    onClick={() => removeFilter(encodedParam)}
    aria-label={`Remove ${class}: ${value} filter`}
    className="ml-1 rounded-full p-1 hover:bg-accent-primary/20 focus-visible:ring-1 focus-visible:ring-accent-primary"
    style={{ minWidth: 44, minHeight: 44 }}
  >
    <X size={10} />
  </button>
</div>
```

Note: the 44×44px touch target is achieved via the `style` override on the remove button. The pill itself is compact; only the interactive region meets the 44px minimum.

**`EntityFilterBar` wrapper:**

```tsx
{activeEntityFilters.length > 0 && (
  <div className="flex flex-wrap items-center gap-2">
    <span className="text-xs text-text-tertiary">Filtering by:</span>
    {activeEntityFilters.map((f) => (
      <EntityFilterPill key={f} encodedParam={f} onRemove={removeFilter} />
    ))}
    {activeEntityFilters.length > 1 && (
      <button
        onClick={clearAllEntityFilters}
        className="text-xs text-text-tertiary hover:text-text-secondary transition-colors"
      >
        Clear all
      </button>
    )}
  </div>
)}
```

**URL param format:** Each active entity filter is one `?entity=` param. Value is `{class}:{value}` percent-encoded. Multi-entity = repeated params. React Router `useSearchParams` drives both read and write.

### 3. SearchPage — "Save this search" Button + Dialog (ENEX-02, D-09)

**Button placement:** Page header, right side, aligned with the page title row:

```tsx
<div className="flex items-center justify-between">
  <div>
    <h1 className="page-title text-text-primary">Search</h1>

  </div>

    <Bookmark size={14} />
    Save this search
  </Button>
</div>
```

- Variant: `secondary` (not `ghost`) — provides a bordered surface that is visible without competing with the search input
- Disabled when: no query AND no active entity filters (nothing meaningful to save)
- Icon: Lucide `Bookmark`, 14px

**Dialog spec (`SaveSearchDialog`):**

```tsx
<Dialog open={open} onOpenChange={onOpenChange}>
  <DialogContent className="sm:max-w-[400px]">
    <DialogHeader>
      <DialogTitle>Save search</DialogTitle>
      <DialogDescription>
        Name this search to find it quickly in the sidebar.
      </DialogDescription>
    </DialogHeader>
    <div className="space-y-4 py-4">
      <div className="space-y-2">
        <Label htmlFor="search-name" className="text-sm font-semibold text-text-primary">
          Name
        </Label>
        <Input
          id="search-name"
          placeholder="e.g. Property Tax 2024"
          value={name}
          onChange={(e) => setName(e.target.value)}

          className="focus-visible:ring-accent-primary"

        className="btn-primary"

        disabled={!name.trim()}
      >
        Save search
      </Button>
    </DialogFooter>

```

- Dialog max width: `sm:max-w-[400px]`
- Save button: `btn-primary` (accent fill) — primary focal point of the dialog
- Keep searching: `ghost` variant
- `Enter` key submits when name field is focused
- Save button disabled when name is empty or whitespace-only

```tsx
    </span>
```

**Each saved search row:**

```tsx
<Link
  to={buildSavedSearchUrl(savedSearch)}
  className={cn(
    "flex items-center gap-3 rounded-md px-3 py-2 text-sm font-medium transition-colors",

      : "text-text-secondary hover:bg-bg-tertiary hover:text-text-primary"

  {!isCollapsed && (

      <span className="text-xs text-text-tertiary flex-shrink-0">
        ({savedSearch.doc_count_cache})
      </span>

```

**Spec:**

- Name: `text-sm font-medium text-text-secondary` → `text-text-primary` on hover/active

- `buildSavedSearchUrl(ss)`: constructs `/search?q={query}&entity={...filters}` from the saved search object

**Loading state:** When `useSavedSearches()` is loading, render 2 skeleton lines (matching Smart Spaces loading skeleton):
```tsx
Array.from({ length: 2 }).map((_, i) => (

    <div className="h-3.5 w-3.5 rounded bg-bg-tertiary animate-pulse flex-shrink-0" />
```

**Empty state (no saved searches):** Section header still renders. Below it: `<p className="px-3 py-2 text-xs text-text-tertiary">No saved searches yet</p>`. The "Save this search" button on SearchPage is the entry point — no link needed here.

**Collapsed sidebar:** Render only the `Bookmark` icon for each saved search row (same as color-dot pattern for spaces). No name, no count.

### Destructive Actions

Phase 11 introduces one destructive action: **deleting a saved search** (sidebar right-click or future delete button on saved search management page). This is deferred to the management surface (out of scope for phase 11 core — only the save flow is in scope). The delete action is tracked here for completeness.

| Action | Destructive? | Confirmation gate? | Rationale |
|--------|-------------|-------------------|-----------|
| "Save this search" | No | None — it's additive | Creates a new entry; nothing removed |
| Remove entity filter pill | No | None | Reversible; user can re-add via chip click |
| "Clear all" entity filters | No | None | Reversible via URL navigation / back button |
| Delete saved search | Mild (removes user-created entry) | Yes — shadcn `AlertDialog` | Permanent deletion; one-step confirmation: "Delete "{name}"? This cannot be undone." Confirm: "Delete search" (destructive variant). Keep searching: "Keep it". |

## Data Type Extensions

Phase 11 adds fields to existing types and introduces `SavedSearch`:

```typescript
// Extension to existing RelatedDocument shape (from useRelatedDocuments hook)
interface RelatedDocument {
  id: string;
  name: string;
  path: string;
  score: number;     // 0.0–1.0 composite score (0.6×cosine + 0.4×entity_jaccard)
  snippet?: string;  // nullable — text excerpt near entity overlap
}

// New type for saved searches
interface SavedSearch {
  id: string;
  name: string;
  query: string;
  filters: {
    entities?: string[];   // Array of "{class}:{value}" strings
    topic?: string | null;
  };
  created_at: string;       // ISO 8601
  doc_count_cache: number;  // Hint for immediate Sidebar render; re-evaluated on mount
}

// Extension to EntityDetailPage data shape
interface EntityPageData {
  class: string;
  value: string;
  aliases: string[];
  total_doc_count: number;
  documents: DocumentRow[];     // paginated 20/page
  co_occurring: EntityRef[];    // top 10
}

interface EntityRef {
  class: string;
  value: string;
  co_doc_count: number;    // how many docs share this entity with the primary entity
}
```
