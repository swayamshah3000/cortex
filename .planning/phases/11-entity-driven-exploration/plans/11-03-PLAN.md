```
pub fn apply_entity_class_filters(
    entity_filters: &[EntityClassFilter],
    entity_store: &EntityStore,
) -> Option<HashSet<String>>
```

Semantics:
- If `entity_filters` is empty → return `None` (no narrowing; identical to the None-case in `apply_metadata_filters`).
- Otherwise iterate filters, look up each `(value.to_lowercase(), class_key)` in `entity_store.alias_index`. For the class key, try the incoming `class` string FIRST, then fall back to `class.to_lowercase()` — this bridges Phase 6 (lowercase entity_type) and Phase 8 (capitalized class name). Per pitfall #5 in 11-RESEARCH.md.
- On any lookup miss → return `Some(HashSet::new())` and terminate the loop (AND-semantics: one missing filter zeroes the intersection).
- On hit → fetch `doc_index[canonical_id]`. First hit initializes the result set; subsequent hits intersect against it. Return `Some(intersection)` at the end.
