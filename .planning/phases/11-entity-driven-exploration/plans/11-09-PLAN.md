```
function buildSavedSearchUrl(ss: SavedSearch): string {
  const params = new URLSearchParams();
  if (ss.query) params.set("q", ss.query);
  (ss.filters.entities ?? []).forEach((e) => params.append("entity", e));
  // Note: topic + docType + spaceId are not URL-driven on SearchPage yet;
  // they live in local state. Round-trip fidelity is entity + query only for Phase 11.
  const qs = params.toString();
  return qs ? `/search?${qs}` : "/search";
}
```
Note: SearchPage query is a local `useState` in Plan 07 — it does NOT read `?q=` from URL. Adding a small `useEffect` in `SearchPage.tsx` to hydrate `query` from `?q=` at mount is out-of-scope for Plan 09.
