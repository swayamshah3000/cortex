## Spacing Scale

Cortex uses a 4px base grid. Inherited from Phase 8 contract. **All phase 9 spacing values are strict multiples of 4. No sub-grid values are declared.**

| Token | Value | Usage in Phase 9 |
|-------|-------|-----------------|
| xs | 4px | Icon-to-text gap in entity hint chip (`gap-1`), entity hint chip vertical padding (`py-1`), shimmer stack gap |
| sm | 8px | Entity hint chip horizontal padding (`px-2`), edit action row gap |
| md | 12px | SpaceLabelingIndicator horizontal padding (`px-3`), card footer section top padding (`pt-3`) |
| lg | 16px | SpaceDetail edit action button row (`gap-4`) |
| xl | 24px | Card padding (inherited `p-6`) |
| 2xl | 32px | Section separator (inherited `space-y-6` on page) |

**Exceptions:**
- Touch targets for icon-only buttons (edit trigger, lock icon region): minimum 44px × 44px wrapper — 44 = 4 × 11, grid-aligned. Applies to the Edit2 pencil trigger button on SpaceDetailPage.

**No sub-grid values.** Phase 9 does NOT use `py-0.5` (2px) or `px-2.5` (10px). Prior draft values were replaced with `py-1` (4px, xs token) and `px-3` (12px, md token) respectively before spec approval.

## Typography

Inherited from Phase 8 contract. Phase 9 introduces no new type roles; all text in new elements maps to existing roles.

| Role | Size | Weight | Line Height | Font | Class | Phase 9 usage |
|------|------|--------|-------------|------|-------|--------------|
| Page title / inline edit | 24px | 600 | 1.2 | Inter | `.page-title` (`text-2xl font-semibold`) | SpaceDetailPage: space name heading, inline edit input inherits same size |
| Section header | 18px | 600 | 1.2 | Inter | `.section-header` (`text-lg font-semibold`) | SpaceCard label: `text-lg font-semibold text-text-primary` |
| Body | 14px | 400 | 1.5 | Inter | `text-sm text-text-secondary` | SpaceDetailPage description text; doc count on SpaceCard |
| Caption / helper | 12px | 400 | 1.4 | Inter | `text-xs text-text-tertiary` | Timestamp on SpaceCard, entity hint chip text, "Generating label…" sub-line, tooltip content |

**Weights declared for phase 9:** exactly **2** — 400 (regular) and 600 (semibold). Matches Phase 8 budget. No 500 (medium) introduced.

**Active sizes in phase 9:** 12px (caption/chip/tooltip), 14px (body/description), 18px (SpaceCard label), 24px (SpaceDetail heading) — 4 sizes. The 24px size is inherited from existing `.page-title` utility; phase 9 does not introduce it.

**Semantic role protection:** 12px is reserved strictly for non-prose display text (chips, timestamps, tooltip content). 14px is reserved for prose (descriptions, doc counts). No future role may occupy the 13px slot — this protects the 2px gap between caption and body as a clean semantic boundary rather than a size-ambiguous cluster.

## Visual Hierarchy

**Primary focal point of SpaceCard (Phase 9 state: generating):** the shimmer skeleton in the label region. When label is generating, the card communicates "in progress" — the shimmer is the eye's first landing point. "Generating label…" sub-line is secondary.

**Primary focal point of SpaceCard (Phase 9 state: ready, user_locked=false):** the space name label (`text-lg font-semibold text-text-primary`) — unchanged from pre-Phase-9 behavior.

**Primary focal point of SpaceCard (Phase 9 state: ready, user_locked=true):** the space name label. The `Lock` icon (12px, `text-text-tertiary`) in the header row is a low-weight indicator — visible on scan but not competing with the label.

**Primary focal point of the SpaceDetail inline edit flow:** the "Save label" button (`btn-primary`, accent fill). Input field, cancel-edit, and clear-override are supporting elements.

**Canonical entity hint chip** is the lowest-weight new element in phase 9. It sits at the bottom of SpaceCard below sample files, uses transparent bg + border-only styling, and must never visually compete with the space name or sample file list. If the card has no sample files, the chip sits immediately below the doc count line.

**Tooltip on SpaceCard hover** supplements but does not replace the card. It must not auto-appear on keyboard focus in a way that obscures the card content. Default Radix Tooltip behavior (300ms open delay, no keyboard trigger on `<Link>`) is correct here.

## Control Specifications

### 1. SpaceCard — Shimmer State (LLML-05, D-14)

**Trigger:** `space.label_status === 'generating'` (field added to Space type; sent via Tauri `space-labeling-progress` event + React Query refetch).

**Label region replacement:**
- Replace `<p className="font-semibold text-text-primary text-lg">{space.name}</p>` with:
  - `<Skeleton className="h-5 w-36" />` — 20px height (matches text-lg 18px at 1.2 line-height), 144px width (matches "Property Tax Records" at semibold 18px)
- Replace `<p className="text-text-tertiary text-sm">{space.documentCount} documents</p>` with:
  - `<Skeleton className="h-4 w-24" />` — 16px height (text-sm at 1.5 line-height), 96px width

**"Generating label…" sub-line:**
- Rendered below the shimmer pair, within the same `<div>` block:
  - `<span className="text-xs text-text-tertiary mt-1">Generating label…</span>` (4px top margin, grid-aligned)

**Sample files while generating:** Render as normal if `space.sampleFiles` has entries. SpaceCard still shows filenames during generation — only the label region shimmers.

**Canonical entity hint chip while generating:** Do NOT render the chip while `label_status === 'generating'`. The hint is derived from the same labeling run; wait for completion.

**Tooltip while generating:** Do NOT render `<TooltipContent>` when `space.description` is null or empty. `<Tooltip>` wrapper stays but `TooltipContent` conditionally renders only when description exists.

### 2. SpaceCard — Description Tooltip (D-16, LLML-02)

**Trigger:** `space.description` is non-null and non-empty string (any length).

**Wrapping pattern:**
```
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger asChild>
      <Link to={`/spaces/${space.id}`} className="card …">
        {/* existing card content */}
      </Link>
    </TooltipTrigger>
    <TooltipContent
      side="bottom"
      sideOffset={4}
      className="max-w-[240px] text-xs leading-relaxed"
    >
      {truncateAt100(space.description)}
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

**Truncation function:**
```typescript
function truncateAt100(text: string): string {
  return text.length > 100 ? text.slice(0, 100) + "…" : text;
}
```
Truncation is done in JavaScript before render — NOT via CSS `truncate` class (which hides overflow without ellipsis control).

**Tooltip visual spec:**
- Max width: `max-w-[240px]` (override shadcn default which has no width constraint)
- Font: `text-xs` (12px, weight 400, Inter)
- Side: `"bottom"` (description appears below the card)
- Side offset: `4` (4px gap between card and tooltip — default from `tooltip.tsx`, do not override)
- Background: `bg-popover` (shadcn default = `--card` = `--bg-secondary` in dark mode, #1A1A24)
- Border: `border border-border-primary` (shadcn default)
- Rounding: `rounded-md` (8px, shadcn default)
- No visible arrow — shadcn Radix Tooltip uses shadow only (no `TooltipArrow` primitive added)
- Open delay: Radix default (300ms) — do not override

### 4. SpaceCard — Canonical Entity Hint Chip (D-17, D-18)

**Trigger:** `space.canonical_entity_hint` is non-null AND `space.label_status !== 'generating'`.

**Format of `space.canonical_entity_hint`:** `"{ClassName}: {value}"` — e.g., `"Property: AlphaComplex"`, `"Person: Alex Doe"`, `"Organization: HDFC Bank"`.

**Placement:** New section at the bottom of the card, below the sample files div, inside the `space-y-3` container:

```tsx
{space.canonical_entity_hint && space.label_status !== 'generating' && (
  <div className="pt-1">
    <EntityHintChip hint={space.canonical_entity_hint} />
  </div>
)}
```

**`EntityHintChip` visual spec:**

Parse `hint` string by splitting on first `": "` to get `[className, value]`.

| Property | Value |
|----------|-------|
| Background | `bg-transparent` |
| Border | `border border-border-secondary` |
| Border radius | `rounded-full` |
| Padding | `px-2 py-1` (8px / 4px — both grid-aligned) |
| Layout | `inline-flex items-center gap-1` |
| Max width | `max-w-[160px]` |
| Icon | Entity-class icon at size 10px, color from entity semantic map |
| Text | `text-xs text-text-tertiary truncate` — full hint string `"{ClassName}: {value}"` |

**Chip height:** 12px text at leading 1.4 = ~17px, plus 4px top + 4px bottom padding = ~25px total. Slightly taller than the prior draft's 18px sub-grid version, but grid-aligned and visually consistent with other chip elements in the design system.

**Entity class icon map for EntityHintChip** (matches Phase 8 EntityChip classes; `Fingerprint text-orange-400` is the catch-all for unmapped classes):

| Class (case-insensitive) | Icon | Color |
|--------------------------|------|-------|
| Person | `Users` | `text-purple-400` |
| Organization | `Building2` | `text-amber-400` |
| Location | `MapPin` | `text-red-400` |
| Date | `Calendar` | `text-blue-400` |
| Amount | `DollarSign` | `text-green-400` |
| Email | `Mail` | `text-cyan-400` |
| Phone | `Phone` | `text-teal-400` |
| Identifier | `Fingerprint` | `text-orange-400` |
| *(any other)* | `Fingerprint` | `text-orange-400` |

**Visual distinction from Phase 8 chips:**

| Chip type | Background | Border | Border radius | Phase |
|-----------|-----------|--------|--------------|-------|
| Topic chip | `bg-accent-primary/10` (filled indigo tint) | `border-accent-primary/20` | `rounded-full` | Phase 8 |
| Tag chip | `bg-bg-tertiary` (filled neutral dark) | `border-border-secondary` | `rounded-md` | Phase 8 |
| Entity hint chip | `bg-transparent` (outline only) | `border-border-secondary` | `rounded-full` | Phase 9 |

The entity hint chip is the only outline-only (transparent bg) chip in the design system. This makes it immediately distinguishable from topic and tag chips even when rendered in close proximity.

**EntityHintChip is non-interactive.** It is a `<div>` (not `<button>` or `<Link>`). Phase 11 may upgrade it to navigate to entity detail.

### 5. SpaceDetailPage — Inline Label Edit (D-15)

**Location:** SpaceDetailPage header section, replacing `<h1 className="page-title text-text-primary">`.

**States:**

| State | Visual |
|-------|--------|
| View (not editing) | `<h1 className="page-title text-text-primary">{space.name}</h1>` + pencil icon trigger (visible on header row hover) |
| Editing | `<Input …>` field replacing the `<h1>`, save/cancel-edit/clear-override action row below |

**View state — edit trigger:**
```tsx
<div className="flex items-center gap-2 group">
  <h1 className="page-title text-text-primary">{space.name}</h1>
  {space.user_locked && <Lock size={14} className="text-text-tertiary" aria-label="Label locked by user" />}
  <button
    onClick={() => setIsEditing(true)}
    className="opacity-0 group-hover:opacity-100 transition-opacity p-1 rounded hover:bg-bg-tertiary"
    aria-label="Edit space label"
    style={{ minWidth: 44, minHeight: 44 }}
  >
    <Edit2 size={16} className="text-text-tertiary" />
  </button>
</div>
```

- Edit trigger icon: Lucide `Edit2`, 16px, `text-text-tertiary`
- Trigger wrapper: 44×44px minimum (touch target), `rounded`, `hover:bg-bg-tertiary`
- Trigger visibility: hidden at rest (`opacity-0`), visible on container hover (`group-hover:opacity-100`), transition 150ms

**Editing state — input:**
```tsx
<Input
  defaultValue={space.name}
  className="text-2xl font-semibold h-auto py-1 px-2 w-full max-w-[400px] bg-bg-secondary border-border-primary focus-visible:ring-accent-primary"
  onKeyDown={(e) => {
    if (e.key === "Enter") handleSave(e.currentTarget.value);
    if (e.key === "Escape") handleCancelEdit();
  }}
  onBlur={(e) => handleSave(e.currentTarget.value)}
  autoFocus
/>
```

- Input font: `text-2xl font-semibold` (matches `.page-title` = 24px/600 — must explicitly override shadcn Input default `text-base/md:text-sm`)
- Input padding: `py-1 px-2` (4px / 8px, grid-aligned)
- Input height: `h-auto` (grows to content, not fixed 40px default)
- Input max width: `max-w-[400px]`
- Focus ring: `focus-visible:ring-accent-primary` (inherited from shadcn Input via `--ring` variable mapping)
- Save triggers: `Enter` key or `onBlur` (clicking outside input)
- Cancel-edit triggers: `Escape` key or click of "Cancel edit" button — reverts input to original label, sets `isEditing=false`

**Editing state — action row:**

```
[Save label button] [Cancel edit button] [Clear override button — only when user_locked]
```

| Button | Variant | Class | Label |
|--------|---------|-------|-------|
| Save | Primary | `btn-primary text-sm px-4 py-2` | "Save label" |
| Cancel edit | Ghost | `btn-ghost text-sm px-4 py-2` | "Cancel edit" |
| Clear override | Ghost (warn hover) | `btn-ghost text-sm px-4 py-2 text-text-tertiary hover:text-destructive` | "Clear override" |

- Action row layout: `flex items-center gap-2 mt-2`
- "Save label" is the primary focal point of the editing state (accent fill, left-most position)
- "Cancel edit" is a verb + noun pair (not generic "Cancel") — describes the specific action of discarding the current edit and reverting to the saved label
- "Clear override" only renders when `space.user_locked === true`
- "Clear override" is NOT confirmation-gated (action is reversible — user can type label back)

**Lock indicator on detail page:** `Lock` icon, 14px (slightly larger than SpaceCard 12px — appropriate for page heading scale), `text-text-tertiary`, placed inline with the heading in the `group` div above.

### 6. SpaceDetailPage — Full Description (D-16, LLML-02)

**Location:** Below the header section (`<div className="flex items-start gap-4">`), above Sub-Spaces or Documents section, inside the `space-y-6` container.

```tsx
{space.description && (
  <p className="text-sm text-text-secondary max-w-prose leading-relaxed">
    {space.description}
  </p>
)}
```

**Spec:**
- Font: 14px, weight 400, `text-text-secondary`
- Line height: `leading-relaxed` (1.625 — slightly more open than `leading-normal` 1.5, appropriate for multi-sentence descriptions)
- Max width: `max-w-prose` (65ch — prevents overly wide text lines on large viewports)
- No truncation: full description rendered (100+ chars is normal; "unwrapped" per requirement means no CSS line-clamp)
- Rendered when: `space.description` is non-null and non-empty
- Not rendered when: description is null, empty string, or space is in `generating` state

**Empty description fallback:** When `space.description` is null/empty AND `space.canonical_entity_hint` is non-null, render:
```tsx
<p className="text-sm text-text-tertiary italic max-w-prose leading-relaxed">
  Space organized around {space.canonical_entity_hint} documents.
</p>
```
Fallback text is `text-text-tertiary italic` — visually muted to distinguish it from a real LLM-generated description.

When both `description` and `canonical_entity_hint` are null/absent: description area is not rendered. No placeholder.

### 7. SpaceDetailPage — "Regenerate label" Button (D-15)

**Location:** SpaceDetailPage header, inline with doc count / stats line:

```tsx
<div className="flex items-center gap-4 mt-1">
  <p className="text-sm text-text-secondary">
    {space.documentCount} documents — Updated {formatRelativeTime(space.lastUpdated)}
  </p>
  <Button
    variant="ghost"
    size="sm"
    onClick={handleRegenerate}
    disabled={isRegenerating || space.label_status === 'generating'}
    className="flex items-center gap-2 text-text-secondary hover:text-text-primary"
  >
    {isRegenerating
      ? <Loader2 size={14} className="animate-spin" />
      : <RefreshCw size={14} />
    }
    {isRegenerating ? "Regenerating…" : "Regenerate label"}
  </Button>
</div>
```

**Spec:**
- Variant: `ghost` — no fill at rest, no destructive color, `hover:bg-bg-tertiary`
- Size: `sm` (shadcn size="sm" = `h-9 px-3 text-sm` — grid-aligned)
- Icon: `RefreshCw` 14px at rest; `Loader2` 14px `animate-spin` when in flight
- Icon-to-text gap: `gap-2` (8px, sm token) — replaces prior draft `gap-1.5` (6px, sub-grid)
- Label: "Regenerate label" (idle) → "Regenerating…" (in flight)
- Disabled states: when `isRegenerating === true` OR when `space.label_status === 'generating'`
- No confirmation gate — re-labeling is non-destructive and cheap to undo (user can rename back)
- `user_locked=true` spaces: Backend skips them; button is still visible but its effect is a no-op that surfaces a toast (see Copywriting §Regenerate no-op)

### 8. SpaceLabelingIndicator — Progress Chip (D-14, LLML-05)

**Location:** SpacesPage header, right side, appearing while a labeling batch is in flight (mirrors BackfillIndicator placement pattern from Phase 8).

**Trigger:** Frontend `useSpaceLabels()` hook subscribes to `space-labeling-progress` Tauri event. Chip appears when `status === 'labeling'`, disappears when `status === 'complete'` (auto-dismiss 3s).

**Visual:**
```tsx
<div className="flex items-center gap-2 px-3 py-1 rounded-full bg-accent-primary/10 text-accent-primary text-xs font-medium">
  <Loader2 size={12} className="animate-spin" />
  Labeling spaces {processed}/{total}
</div>
```

**Grid-aligned spacing:**
- Horizontal padding: `px-3` (12px, md token) — grid-aligned. Phase 9 does NOT reuse `BackfillIndicator`'s `px-2.5` (10px) sub-grid pattern; the labeling indicator sets a cleaner baseline for future components.
- Vertical padding: `py-1` (4px, xs token) — grid-aligned
- Icon-to-text gap: `gap-2` (8px, sm token) — grid-aligned
- Total chip height: 12px text at leading 1 + 4px top + 4px bottom = ~20px

**States:**

| Status | Visual | Duration |
|--------|--------|---------|
| `labeling` | Spinner + "Labeling spaces X/Y" | While in flight |
| `complete` | "Labels ready" (no spinner, success green) | Auto-dismiss 3s |
| `error` | "Labeling failed" (error red) | Stays until dismissed |

- `complete` color: `bg-success/10 text-success`
- `error` color: `bg-error/10 text-error`

## Copywriting Contract

### Primary CTA

| Context | CTA Label |
|---------|-----------|
| SpaceDetail inline edit — save action | "Save label" |
| SpaceDetail — regenerate action (idle) | "Regenerate label" |
| SpaceDetail — regenerate action (in flight) | "Regenerating…" |
| SpaceDetail — clear user lock | "Clear override" |
| SpaceDetail — discard current edit | "Cancel edit" |
| SpaceCard — generating state sub-line | "Generating label…" |

**Note:** "Cancel edit" (not bare "Cancel") is intentional. The verb + noun pair specifies which action is being cancelled — the current label edit — and avoids the generic-CTA anti-pattern that "Cancel" would introduce alongside a `SpaceLabelingIndicator` or `Regenerate label` action that could also be plausibly cancelled.

### Destructive Actions

No fully destructive actions in Phase 9. Analysis:

| Action | Destructive? | Confirmation gate? | Rationale |

"Clear override" hover-into-destructive color (`hover:text-destructive`) is a visual affordance only — it does not gate the action with a confirmation dialog.

## Data Type Extensions

Phase 9 adds three fields to the `Space` type in `client/lib/types.ts`:

```typescript
interface Space {
  // ... existing fields ...
  description?: string;                          // LLM-generated 1-sentence description (nullable)
  user_locked?: boolean;                         // true when user has manually renamed this space
```

**Frontend handling for missing fields:** Treat `label_status` as `'ready'` when the field is absent (backwards compatibility with spaces not yet touched by Phase 9 labeler). Treat `user_locked` as `false` when absent. Treat `description` and `canonical_entity_hint` as `null` when absent.
