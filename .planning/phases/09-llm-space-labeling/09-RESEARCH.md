## Summary

Phase 9 adds LLM-generated labels and descriptions to Smart Spaces, along with a fingerprint-based cache that prevents re-labeling on unchanged clusters. The primary implementation work is in Rust: a new `spaces/llm_labeler.rs` module that calls `ai_request()` (the Phase 7 provider-agnostic router), a `spaces/label_cache.rs` that reads/writes `space_labels.json`, and a `spaces/fingerprint.rs` that computes SHA-256 membership fingerprints. Frontend changes extend `SpaceCard` with a shimmer state and Radix Tooltip, extend `SpaceDetailPage` with an inline label editor, and add a `SpaceLabelingIndicator` progress chip.

**CRITICAL FINDING:** Both `ruvector-cluster` and `ruvector-domain-expansion` are the wrong crates for what the CONTEXT.md describes. `ruvector-cluster` is a distributed network cluster manager (gossip protocol, shard distribution, consistent hashing) — it has no HDBSCAN or document vector clustering capability. `ruvector-domain-expansion` is a meta-learning framework using Thompson Sampling for AI task policy transfer (Rust synthesis, planning, tool orchestration) — it has no cosine-similarity-based label transfer between vector clusters. The planner must route around these two locked decisions.

**Primary recommendation:** Skip `ruvector-cluster` and `ruvector-domain-expansion`. Keep the existing k-means in `clustering.rs` (D-12 fallback is the right path — the fallback IS the current state). Implement domain expansion label bootstrap as a pure Rust cosine-similarity computation (identical pattern to `manager.rs` lines 126–157, threshold raised to 0.75). No new crate dependencies needed for the core Phase 9 features.

## Common Pitfalls

### Pitfall 1: Adding ruvector-cluster or ruvector-domain-expansion as dependencies

**What goes wrong:** `cargo check` may succeed (both crates compile), but neither crate provides the HDBSCAN or label-transfer API described in the CONTEXT.md. Code that tries to call a non-existent API will fail to compile.
**Why it happens:** The CONTEXT.md locked decisions were written based on incorrect assumptions about these crates' functionality.
**How to avoid:** Do NOT add `ruvector-cluster` or `ruvector-domain-expansion` to `Cargo.toml`. Keep k-means; implement domain expansion bootstrap as pure cosine similarity.
**Warning signs:** If you see `ruvector_cluster::Hdbscan` or `ruvector_domain_expansion::transfer_label()` in a plan task — those symbols do not exist.

### Pitfall 2: Deleting naming.rs prematurely

**What goes wrong:** LLM providers can be disconnected; on all-provider-fail the labeler needs a rule-based fallback. If naming.rs is deleted, spaces get no label at all.
**Why it happens:** naming.rs looks obsolete once llm_labeler.rs exists.
**How to avoid:** Keep naming.rs. Mark `#[allow(dead_code)]` if needed. Wire it as fallback only when `llm_labeler` returns an error after retries. Full deletion belongs in Phase 12+ cleanup.
**Warning signs:** Success criterion 6 ("clustering runs through ruvector-cluster") from ROADMAP.md is technically unachievable with the actual ruvector-cluster crate — treat it as "clustering still runs, just via k-means".

### Pitfall 3: Space struct immutability — forgetting to extend types.rs and frontend types

**What goes wrong:** New fields (`description`, `user_locked`, `canonical_entity_hint`, `label_status`) added to the Rust `Space` struct but not added to the TypeScript `Space` interface in `client/lib/types.ts` — or vice versa. Serde camelCase mismatch causes fields to silently become `undefined` in the frontend.
**Why it happens:** Two separate type systems must be kept in sync manually.
**How to avoid:** Update `src-tauri/src/types.rs` Space struct AND `client/lib/types.ts` Space interface in the same commit. Use `#[serde(default)]` on all new optional fields so existing RPC calls don't break.
**Warning signs:** `space.description` logs as `undefined` in frontend console even when Rust emits it.

### Pitfall 4: Fingerprint collisions across different Space IDs

**What goes wrong:** Two different spaces happen to have the same document membership (same doc-ids, different space-ids). The fingerprint collides. The cache entry for one space gets overwritten by the other.
**Why it happens:** The fingerprint key is the doc-id set hash, not the space-id. If two spaces are identical (possible after manual moves), their fingerprints match.
**How to avoid:** Key the cache by `space_id`, not by fingerprint alone. Use the fingerprint WITHIN the cache entry to check if re-labeling is needed. Cache schema: `{ [space_id]: { fingerprint, label, ... } }`.
**Warning signs:** Space A's label appearing on Space B after recluster.

### Pitfall 5: `label_status` field missing from SpaceManager — frontend gets stuck in shimmer

**What goes wrong:** Frontend SpaceCard reads `space.label_status` to decide shimmer state. If SpaceManager emits Space objects without this field, `label_status` is `undefined`, which the UI-SPEC says to treat as `'ready'`. But if the field IS present as `'generating'` and the backend never emits a `complete` event, the card stays in shimmer forever.
**Why it happens:** The Tauri event `space-labeling-progress` with `status: 'complete'` may be dropped if the labeling task panics.
**How to avoid:** In the labeling task, always wrap the entire batch in a `catch`; emit `status: 'error'` even on panic. In `AppState`, track `labeling_in_progress: bool` and reset on drop.
**Warning signs:** SpaceCard stuck showing "Generating label…" after app restart.

### Pitfall 6: LLM collision retry sends the wrong prompt

**What goes wrong:** After detecting a collision, the retry prompt sends the full SPACE_LABEL_PROMPT but forgets to include "Avoid these labels: [...]". The LLM generates the same label again.
**Why it happens:** The collision retry is a different code path from the initial label call; easy to miss appending the avoid-list to the user message.
**How to avoid:** Extract a `label_with_avoid_list(avoid: &[String])` function that appends the avoid-list to the user message before calling `ai_request_with_retry`.

## Code Examples

### Fingerprint computation [ASSUMED — pattern based on sha2 crate API + Cargo.toml verification]

```rust
// spaces/fingerprint.rs
use sha2::{Sha256, Digest};
use std::collections::HashSet;

/// Compute a 16-char hex fingerprint from a cluster's document ID set.
/// Sorted to be order-independent: same members → same fingerprint.
pub fn membership_fingerprint(doc_ids: &[String]) -> String {
    let mut sorted: Vec<&String> = doc_ids.iter().collect();
    sorted.sort();
    let mut hasher = Sha256::new();
    for id in sorted {
        hasher.update(id.as_bytes());
        hasher.update(b"\n"); // separator to avoid "ab"+"c" == "a"+"bc"
    }
    let result = hasher.finalize();
    format!("{:x}", result)[..16].to_string()
}

/// Jaccard distance: |added ∪ removed| / |union| (D-06).
pub fn jaccard_distance(old_ids: &HashSet<String>, new_ids: &HashSet<String>) -> f32 {
    let added = new_ids.difference(old_ids).count();
    let removed = old_ids.difference(new_ids).count();
    let union_size = old_ids.union(new_ids).count();
    if union_size == 0 { return 0.0; }
    (added + removed) as f32 / union_size as f32
}
```

### space_labels.json sidecar (D-07) [ASSUMED — mirrors settings.rs pattern]

```rust
// spaces/label_cache.rs
use serde::{Deserialize, Serialize};
use std::collections::HashMap;
use std::path::PathBuf;

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct SpaceLabelEntry {
    pub fingerprint: String,
    pub label: String,
    pub description: String,
    pub canonical_entity_hint: Option<String>,
    pub generated_at: String,
    pub user_locked: bool,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct SpaceLabelCache {
    pub labels: HashMap<String, SpaceLabelEntry>,
}

impl SpaceLabelCache {
    pub fn load(app_data_dir: &PathBuf) -> Self {
        let path = app_data_dir.join("space_labels.json");
        std::fs::read_to_string(&path)
            .ok()
            .and_then(|s| serde_json::from_str(&s).ok())
            .unwrap_or_default()
    }

    pub fn save(&self, app_data_dir: &PathBuf) -> Result<(), std::io::Error> {
        let path = app_data_dir.join("space_labels.json");
        let json = serde_json::to_string_pretty(self)?;
        std::fs::write(path, json)
    }
}
```

    pub label: String,
    pub description: String,
}

pub async fn label_cluster(
    auth: &AuthState,
    model: &str,
    doc_titles: &[String],         // top-20 nearest centroid
    entity_summary: &str,           // "Person: 12, Organization: 8, Amount: 5, Date: 42"
    top_topics: &[String],          // top-3 from Phase 8 topic field
    top_tags: &[String],            // top-10 from Phase 8 tags field
    doc_count: usize,
    avoid_labels: &[String],        // for collision retry (D-13)
) -> Result<SpaceLabel, String> {
    let titles_text = doc_titles.iter().take(20)
        .enumerate()
        .map(|(i, t)| format!("{}. {}", i+1, t))
        .collect::<Vec<_>>().join("\n");

    let avoid_text = if avoid_labels.is_empty() {
        String::new()
    } else {
        format!("\n\nIMPORTANT: Avoid these labels already in use: {}",
            avoid_labels.join(", "))
    };

    let user_content = format!(
        "Cluster size: {} documents\n\nTop document titles:\n{}\n\nEntity summary: {}\nTop topics: {}\nTop tags: {}{}",
        doc_count,
        titles_text,
        entity_summary,
        top_topics.join(", "),
        top_tags.join(", "),
        avoid_text
    );

    let req = AIServiceRequest {
        system_prompt: SPACE_LABEL_PROMPT.to_string(),
        messages: vec![ServiceMessage { role: "user".to_string(), content: user_content }],
        max_tokens: Some(256),
        temperature: Some(LABEL_TEMPERATURE),
        response_format: None,
    };

    let response = ai_request_with_retry(auth, req, MAX_RETRIES)
        .await
        .map_err(|e| format!("LLM label call failed: {}", e))?;

    let stripped = strip_json_fences(&response.content);
    serde_json::from_str::<SpaceLabel>(&stripped)
        .map_err(|e| format!("LLM JSON parse failed: {}", e))
}

/// Bootstrap a label for a new cluster from the nearest existing labeled space.
/// Returns the transferred label string if cosine similarity >= 0.75 (D-11).
/// Otherwise returns None → caller triggers full LLM label call.
pub fn try_bootstrap_from_nearest(
    new_centroid: &[f32],
    cached_spaces: &[(String, String, Vec<f32>)], // (space_id, label, centroid)
) -> Option<String> {
    let mut best_sim = f32::NEG_INFINITY;
    let mut best_label: Option<&str> = None;

    for (_, label, centroid) in cached_spaces {
        let sim = cosine_similarity(new_centroid, centroid);
        if sim > best_sim {
            best_sim = sim;
            best_label = Some(label.as_str());
        }
    }

    if best_sim >= 0.75 {
        best_label.map(String::from)
    } else {
        None
    }
}
