  <name>Task 1: Fingerprint module (SHA-256 + Jaccard)</name>
  <files>src-tauri/src/spaces/fingerprint.rs, src-tauri/src/spaces/mod.rs</files>
  <behavior>
    - `membership_fingerprint(&["b","a","c"])` == `membership_fingerprint(&["a","b","c"])` (order-independent, D-05)
    - Returns exactly 16 hex chars (D-05: first 16 hex chars of SHA-256)
    - Separator between IDs prevents "ab"+"c" == "a"+"bc" collision (verified by 3-way test)
    - `jaccard_distance(&{a,b,c}, &{a,b,c})` == 0.0
    - `jaccard_distance(&{a,b,c,d}, &{a,b,c})` == 1/4 == 0.25 (one removed, four in union → above 0.20 threshold)
    - `jaccard_distance(&{a,b,c,d,e}, &{a,b,c,d})` == 1/5 == 0.20 (borderline — exactly at threshold, NOT above)
    - `jaccard_distance(&{}, &{})` == 0.0 (empty union guard, not divide-by-zero)
    - `jaccard_distance(&{a}, &{b})` == 2/2 == 1.0 (fully disjoint = distance 1)
  </behavior>
  <action>
Create `src-tauri/src/spaces/fingerprint.rs` implementing exactly two public functions per the D-05 / D-06 formulas and the code sketch in 09-RESEARCH.md §Code Examples "Fingerprint computation":

- `pub fn membership_fingerprint(doc_ids: &[String]) -> String` — collects references, sorts (Vec<&String> sorted by &str), pushes each id + a `b"\n"` separator into `Sha256::new()`, finalizes, hex-formats via `format!("{:x}", result)`, slices to 16 chars.
- `pub fn jaccard_distance(old_ids: &HashSet<String>, new_ids: &HashSet<String>) -> f32` — returns `0.0` when union size is 0 (guards divide-by-zero), else `(added_count + removed_count) as f32 / union_size as f32`.
