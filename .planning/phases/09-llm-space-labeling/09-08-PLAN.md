  <how-to-verify>
Run the following verification session — every step must pass before typing "approved":

**Preconditions:**
1. `cargo build --release -p cortex` clean.
2. `bun run --cwd . build` clean (or dev server up).
3. At least one AI provider connected in Settings → AI (Anthropic, OpenAI, Gemini, or Ollama).
4. A watched folder with a diverse set of documents (mixed topics — e.g., 20+ docs across property, kids, finance, medical, work, receipts). Suggested: `~/private` sample corpus per 09-CONTEXT.md §Specifics.

**Step 1 — LLML-01: LLM-generated labels appear:**
- Navigate to /spaces.
- Click "Re-cluster" in the top-right.
- Watch: SpaceLabelingIndicator chip appears in the header ("Labeling spaces X/Y" with spinner).
- Watch: individual SpaceCards flip from shimmer state to real labels one by one.
- After batch completes: verify at least 3 different spaces have labels that are corpus-specific (e.g., "Property Tax Records", "Kids School Docs", not "Space 1" or "Contacts & Correspondence"). Screenshot the /spaces grid for the SUMMARY.

**Step 2 — LLML-02: Description tooltip + detail prose:**
- Hover any labeled SpaceCard. A dark tooltip appears below the card showing the description (≤ 100 chars + ellipsis if longer). Screenshot.
- Click into that space (open /spaces/:id).
- Verify the full description (no truncation) renders below the header, styled `text-sm text-text-secondary max-w-prose leading-relaxed`.

**Step 3 — LLML-03: Fingerprint cache skip:**
- With no docs added/removed, click "Re-cluster" again.
- Watch: the SpaceLabelingIndicator processed count should show most/all spaces skipping the LLM (progress races through them because they hit the cache).
- Confirm: labels + descriptions are unchanged from Step 1.
- Look at `~/Library/Application Support/{app-id}/space_labels.json` (or the OS equivalent) — the file exists and contains entries keyed by space_id.

**Step 4 — LLML-04: No duplicate labels:**
- After Step 1's batch: scan /spaces grid. Verify no two spaces share a label.
- (Optional stress test) Trigger recluster on a corpus that would tend to produce duplicates (e.g., many work docs). Verify duplicates get either LLM-retry with distinct labels OR suffix-fallback (e.g., "Work Docs — Freelance").

**Step 5 — LLML-05: Background + navigable:**
- Start a re-cluster.
- WHILE the batch is running: navigate to /search, /recent, /favorites, /tags, /watched, /settings. Every page should be reachable and interactive.
- Return to /spaces. Confirm SpaceLabelingIndicator is still ticking; SpaceCards not-yet-labeled show shimmer + "Generating label…".

**Step 6 — D-15: User lock UX:**
- On /spaces/:id: hover the header → Edit2 pencil appears → click it.
- Type a new label (e.g., "My Custom Label") → press Enter.
- Toast reads verbatim: `Label saved and locked. Cortex won't overwrite this name.`
- Header shows the new label + a Lock icon.
- Return to /spaces — SpaceCard for that space shows the Lock icon in the header row.
- Click "Regenerate label" on the detail page.
- Toast reads verbatim: `Label for "My Custom Label" is locked. Clear the override first to allow regeneration.`
- Click "Clear override".
- Toast reads verbatim: `Override cleared. This space will be re-labeled on next recluster.`
- Verify Lock icon is now gone.
- Click "Regenerate label" again — this time it should proceed (spinner + "Regenerating…" text), and after ~5-10s the label updates.

**Step 7 — D-17 / D-18: Canonical entity hint chip:**
- Find a space where a single entity dominates (e.g., property docs where "Property: AlphaComplex" appears in > 20% of docs).
- Verify SpaceCard shows the EntityHintChip below sample files: outline-only pill with entity-class icon (Home / Users / Building2 etc.) + `"ClassName: value"` text.
- Verify a mixed space (no dominant entity) does NOT show the chip.

**Step 8 — SC6 deviation acknowledgement:**
- Confirm `Cargo.toml` was NOT modified — grep for `ruvector-cluster` or `ruvector-domain-expansion` returns nothing new. Per 09-RESEARCH.md, these crates don't provide HDBSCAN / centroid label transfer, so k-means was retained. SC6 is met "in spirit" (clustering still runs correctly) but not "in letter" (ruvector-cluster is not the backend). Document the deviation in the SUMMARY.

**Any failures?**
Describe each failure with the step number, expected vs actual behavior, and a screenshot if possible. Blocking failures → /gsd:plan-phase 09 --gaps to generate closure plans. Non-blocking observations → capture in the SUMMARY for later polish.
