Existing lib.rs `.setup(|app| { ... })` closure loads `saved_search_store` via `SavedSearchStore::load(&app_data_dir)` then wraps in `Arc::new(Mutex::new(...))`. Mirror this exact pattern for `TripleStore`.

Existing IPC pattern (`src-tauri/src/saved_searches/commands.rs`):
```rust
#[tauri::command]
pub async fn get_saved_searches(state: State<'_, AppState>) -> Result<Vec<SavedSearch>, AppError> {
    let store_arc = state.saved_search_store.clone();
    let store = store_arc.lock().await;
    Ok(store.all().to_vec())
}
```
Wrap sync std::sync::Mutex work in `tokio::task::spawn_blocking` when needed; tokio::sync::Mutex can be `await`ed directly.

TripleStore lock choice: use `Arc<tokio::sync::Mutex<TripleStore>>` (mirror of SavedSearchStore) since commands hold the lock across `store.save(&app_data_dir)` I/O and across `spawn_blocking` boundaries. This matches how backfill acquires the lock via `blocking_lock`.

Types (Plan 01):
- `Triple, TripleWithEntities, RelationsPageData, OwnershipPageData, AssetType, PredicateSubjectPair, PredicateObjectPair`
- `is_valid_predicate`

AssetType derivation rules (from D-14, D-18):
- `AssetType::Property` when `object.entity_type == "location"` AND object canonical is linked to a doc whose topic ∈ {"property", "real_estate"}. Fallback: check `object.canonical_name` for "plot", "property" hints.
- `AssetType::Vehicle` when object.entity_type == "location" AND topic == "vehicle" (per D-18 — vehicles surface as Location in Phase 8 output because model names look like place names).
- `AssetType::Investment` when object topic ∈ {"investment", "insurance"}.
- `AssetType::Business` when object.entity_type == "organization" AND topic == "business".
- `AssetType::Financial` when object.entity_type == "identifier" AND subclass ∈ {"bank_account", "iban", "policy_number", "folio_number", "credit_card"}.
- `AssetType::Other` otherwise.

Since triples themselves don't carry topic (topic is on doc metadata), the classifier needs to inspect the OBJECT entity's `entity_type`/`subclass` AND look up one representative doc's topic via `triple.doc_ids[0]` → engine → metadata.topic. Keep the classifier a private helper in `commands/relations.rs`.

Helper (private): `fn expand_triple(triple: &Triple, entity_store: &crate::graph::entity_store::EntityStore) -> Option<TripleWithEntities>` — look up subject + object CanonicalEntity from EntityStore.canonicals; return None if either is missing (log warning).

Helper (private): `fn classify_asset_type(object: &CanonicalEntity, triple: &Triple, engine: &crate::engine::CortexEngine) -> AssetType` — implements D-14 / D-18 asset classification. Reads doc topic from the first doc in `triple.doc_ids` (if any) via `engine.collections.get_collection("documents_384")` → `db.get(&doc_ids[0])` → metadata.topic. Logic:
- If object.entity_type == "identifier": inspect subclass on any ExtractedEntity in the object canonical's alias set for {"bank_account", "iban", "policy_number", "folio_number", "credit_card"} → AssetType::Financial; else Other.
- If object.entity_type == "location":
  - topic == "vehicle" → AssetType::Vehicle
  - topic in {"property", "real_estate"} → AssetType::Property
  - fallback: contains "plot"/"land"/"property" in canonical_name → Property; contains vehicle model hints → Vehicle; else Other.
- If object.entity_type == "organization":
  - topic == "business" → AssetType::Business
  - fallback Other.
- topic in {"investment", "insurance"} → AssetType::Investment (regardless of entity_type — overrides above).
- Default → AssetType::Other.

Command 1: `get_entity_relations(entity_id: String) -> Result<RelationsPageData, AppError>` (D-13).
- spawn_blocking to acquire triple_store and entity_store locks.
- Look up canonical via entity_store.get_canonical(&entity_id) → NotFound if absent.
- Get all triples via triple_store.get_by_entity(&entity_id).
- Partition into outgoing (subject_id == entity_id) and incoming (object_id == entity_id).
- Expand each Triple to TripleWithEntities via helper.
- Return RelationsPageData { entity, outgoing, incoming }.

Command 2: `get_all_owned_by(person_id: String) -> Result<OwnershipPageData, AppError>` (D-14).
- Look up person canonical → NotFound if absent.
- Query forward index: triple_store.get_objects_for(&person_id, "owns") — returns object_ids for triples where person is subject and predicate=="owns".
- For each object_id: find the underlying Triple (search all_by_entity for the matching object) and the CanonicalEntity for that object.
- Group by classify_asset_type(object, triple, engine).
- Return OwnershipPageData { person, assets_by_type: HashMap<AssetType, Vec<TripleWithEntities>>, total_assets }.

Command 3: `get_all_related_to(entity_id: String) -> Result<Vec<(String, CanonicalEntity)>, AppError>` (D-15) — flat list of (predicate, other entity) for every triple touching entity_id (both directions).
- spawn_blocking; look up entity; iterate triple_store.get_by_entity(&entity_id).
- For each triple where entity_id == subject_id, push (triple.predicate.clone(), object_canonical).
- For each triple where entity_id == object_id, push (triple.predicate.clone(), subject_canonical).
- Deduplicate by (predicate, other.id).

Command 4: `get_subjects_by_predicate_object(predicate: String, object_id: String) -> Result<Vec<PredicateSubjectPair>, AppError>` — for "who owns AlphaComplex?" type queries.
- Validate predicate via is_valid_predicate → BadRequest if invalid.
- triple_store.get_subjects_for(&predicate, &object_id) → subject_ids.
- For each subject_id: look up CanonicalEntity and the specific Triple (linear scan of triple_store.all() with matching (subject_id, predicate, object_id)).
- Return Vec<PredicateSubjectPair>.

Command 5: `get_objects_by_subject_predicate(subject_id: String, predicate: String) -> Result<Vec<PredicateObjectPair>, AppError>` — "what does Alex own?" type queries.
- Mirror of command 4 with forward index.

Command 6: `add_manual_triple(subject_id: String, predicate: String, object_id: String, doc_id: Option<String>) -> Result<Triple, AppError>` (D-16).
- Validate: subject_id != object_id, is_valid_predicate(&predicate), both entity ids exist in entity_store.
- spawn_blocking: acquire triple_store lock; call `triple_store.add_manual(subject_id, predicate, object_id, doc_id)` — returns the new Triple.id.
- Save triple_store to app_data_dir immediately (mutating command → persist now).
- Return the newly inserted Triple (fetch via triple_store.get(&id)).

Command 7: `delete_triple(triple_id: String) -> Result<(), AppError>` (D-16).
- spawn_blocking: acquire triple_store lock; call `triple_store.delete(&triple_id)`.
- If returns false → return `Err(AppError::NotFound(format!("triple not found: {}", triple_id)))`.
- Save triple_store to app_data_dir immediately.
- Return Ok(()).
