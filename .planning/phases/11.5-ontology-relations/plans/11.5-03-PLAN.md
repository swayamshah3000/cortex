- `pub const EXTRACT_RELATIONS_PROMPT: &str` ‚Äî inline `r#"..."#` string containing the full system prompt. The prompt MUST include:
  - Role: "You extract relations (triples) from a document."
  - 8 explicit rules:
    1. Output ONLY valid JSON matching schema. No fences, no preamble, no explanation.
    2. Predicates are FIXED. Choose ONLY from this vocabulary: `owns, owned_by, purchased_from, sold_to, located_in, part_of, address_of, married_to, parent_of, child_of, member_of, employer_of, employee_of, partner_of, issued_by, dated, signed_by, uses_pan, uses_aadhaar, has_voter_id, mentioned_with`.
    3. If a relation doesn't fit any predicate, emit `mentioned_with` (weak default) or omit the triple entirely. NEVER invent new predicates.
    4. Subject and object IDs MUST reference the entity list (e.g. `e_0`, `e_1`) ‚Äî do NOT invent free-form names.
    5. Only emit triples the document TEXT actually supports. No speculation.
    6. Prefer directional predicates (`A owns B`) over symmetric (`A mentioned_with B`). Do not emit both directions ‚Äî the store auto-inverts `owns`/`owned_by`, etc.
    7. The document may be OCR'd; tolerate typos but do not fabricate relationships to fix them.
    8. Output an empty `triples` array `[]` when no relations are clearly supported.
  - JSON schema (camelCase, matching Pass3Output below):
    ```
    { "triples": [ { "subjectId": "e_N", "predicate": "owns", "objectId": "e_M" }, ... ] }
    ```
  - 4 multi-region few-shot examples (following Pass 2 pattern ‚Äî India/vehicle/property/finance mixed):
    Example A: Property sale deed
      Entities: e_0 Person "Alex Doe", e_1 Location "AlphaComplex", e_2 Location "Pune", e_3 Date "2024-04-11"
      Text excerpt: "This Sale Deed dated 11 April 2024 confirms that Alex Doe has purchased AlphaComplex located in Pune from John Roe."
      ‚Üí `{"triples":[{"subjectId":"e_0","predicate":"owns","objectId":"e_1"},{"subjectId":"e_1","predicate":"located_in","objectId":"e_2"},{"subjectId":"e_0","predicate":"purchased_from","objectId":"e_4"},{"subjectId":"e_0","predicate":"dated","objectId":"e_3"}]}`
    Example B: Vehicle registration
      Entities: e_0 Person "John Roe", e_1 Location "Unit 204", e_2 Identifier "MH12AB1234"
      Text: "Owner: John Roe. Vehicle: Unit 204. Registration No: MH12AB1234."
      ‚Üí `{"triples":[{"subjectId":"e_0","predicate":"owns","objectId":"e_1"},{"subjectId":"e_1","predicate":"has_voter_id","objectId":"e_2"}]}`
      (Intentionally shows omission over invention: no `registered_as` predicate exists in vocabulary, so the extractor picks the closest available; add a comment in prompt: "prefer omission over stretching a predicate.")
    Example C: Identity doc
      Entities: e_0 Person "Jane Doe", e_1 Identifier "ABCDE1234F"
      Text: "Name: Jane Doe. PAN: ABCDE1234F."
      ‚Üí `{"triples":[{"subjectId":"e_0","predicate":"uses_pan","objectId":"e_1"}]}`
    Example D: Empty case
      Entities: e_0 Amount "$500"
      Text: "Miscellaneous receipt total $500."
      ‚Üí `{"triples":[]}`
  - Closing line: "OUTPUT ONLY THE JSON ‚Äî NO ADDITIONAL TEXT."

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cortex ‚Üí LLM provider HTTP | Doc text + entity list crosses to third-party API |
| LLM response ‚Üí parse | Untrusted JSON string parsed into `Pass3Output` |
| Pass3Output ‚Üí TripleStore | Extracted triples cross into persistent state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11.5-09 | Tampering | LLM emits fabricated relations not present in text | mitigate | Prompt Rule 5 requires text-supported triples; validate_and_normalize drops out-of-bounds entity references; user override IPC (Plan 04) lets user delete false triples |
| T-11.5-10 | Tampering | LLM invents new predicate | mitigate | Prompt Rules 2+3 explicitly lock the vocabulary; `is_valid_predicate` guard in validate_and_normalize drops unknown predicates |
| T-11.5-11 | Information Disclosure | Doc text sent to third-party LLM | accept | Same trust model as Pass 2 ‚Äî user consented to LLM extraction via Settings toggle (Phase 8 D-33). No new disclosure surface |
| T-11.5-12 | Denial of Service | Runaway LLM requests exhaust rate limits | mitigate | tokio::sync::Semaphore cap 8 (D-08); ai_request_with_retry respects provider Retry-After (Phase 7); one Pass 3 call per doc (D-05 pattern mirror) |
| T-11.5-13 | Tampering | JSON parse succeeds but yields self-referential or duplicate triples | mitigate | validate_and_normalize drops self-referential + out-of-bounds triples; TripleStore.insert_one further validates + dedupes by (subject, predicate, object) |
</threat_model>
