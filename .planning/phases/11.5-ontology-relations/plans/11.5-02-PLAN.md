Imports: `crate::types::{Triple, AUTO_INVERSE_PAIRS, SYMMETRIC_PREDICATES, is_valid_predicate}`, `std::collections::{HashMap, HashSet}`, `std::path::Path`, `serde::{Serialize, Deserialize}`, `uuid::Uuid`.

Declare `pub struct TripleStore` with fields:
- `pub triples: HashMap<String, Triple>` — canonical map keyed by `triple.id`
- `#[serde(skip)] pub forward_index: HashMap<(String, String), HashSet<String>>` — (subject_id, predicate) → set of object_ids (rebuilt after load)
- `#[serde(skip)] pub reverse_index: HashMap<(String, String), HashSet<String>>` — (predicate, object_id) → set of subject_ids (rebuilt after load)
- `#[serde(skip)] pub entity_touches: HashMap<String, HashSet<String>>` — entity_id → set of triple_ids where entity is subject OR object (rebuilt after load; enables fast `get_entity_relations`)

Derive `Debug, Clone, Serialize, Deserialize, Default` on the struct and add `#[serde(rename_all = "camelCase")]` on the outer struct.

**Persistence format:** Only `triples` field is serialized (indices are `#[serde(skip)]` — reconstructed by `rebuild_indices()` after load). JSON schema:
```json
{ "triples": { "t-uuid-1": { "id": "t-uuid-1", "subjectId": "...", "predicate": "owns", "objectId": "...", "docIds": ["d1"], "userAdded": false, "createdAt": "2026-07-08T10:00Z" }, ... } }
```

Implement these methods on `impl TripleStore`:

1. `pub fn load(app_data_dir: &Path) -> Self` — read `app_data_dir/triples.json` via `std::fs::read_to_string`; on any error return `Self::default()`. On success, call `rebuild_indices()` before returning. Never panics (mirror T-11-04 resilience contract from SavedSearchStore).

2. `pub fn save(&self, app_data_dir: &Path) -> std::io::Result<()>` — write `triples.json` via `serde_json::to_string_pretty` + `std::fs::write`. Create parent dir if needed. Overwrites atomically per POSIX semantics.

3. `pub fn rebuild_indices(&mut self)` — clear + repopulate `forward_index`, `reverse_index`, `entity_touches` from `self.triples`. Called by `load()` and by tests after mutating `triples` directly.

4. `pub fn get(&self, triple_id: &str) -> Option<&Triple>` — direct HashMap lookup.

5. `pub fn all(&self) -> impl Iterator<Item = &Triple>` — iterate all triples.

6. `pub fn insert_one(&mut self, mut triple: Triple) -> Result<String, String>` — internal helper. Validates:
   - `is_valid_predicate(&triple.predicate)` — else `Err(format!("invalid predicate: {}", triple.predicate))`
   - `triple.subject_id != triple.object_id` — else `Err("subject_id must differ from object_id")`
   Assigns fresh id if `triple.id.is_empty()`: `format!("t-{}", Uuid::new_v4())`.
   Assigns `triple.created_at` if empty (RFC-3339 UTC via `chrono::Utc::now().to_rfc3339()`).
   Upsert semantics: if a triple with the same (subject_id, predicate, object_id) already exists AND stored has `user_added == true`, do NOT overwrite user_added — instead merge `doc_ids` into the existing user-added triple and return its id (D-12: user overrides preserved across LLM re-runs).
   Otherwise: replace/insert into `triples`, update all three indices, return the assigned id.

7. `pub fn upsert_from_doc(&mut self, doc_id: &str, mut incoming: Vec<Triple>) -> Result<Vec<String>, String>` — called by Pass 3 backfill. For each incoming triple:
   - Set `t.doc_ids = vec![doc_id.to_string()]` (Pass 3 provides the doc scope).
   - Set `t.user_added = false`.
   - Look for an existing triple with matching (subject_id, predicate, object_id): if found AND `existing.user_added == true`, skip (do not clobber user override) but still merge doc_id into `existing.doc_ids` (unique-append) — D-12.
   - Otherwise `insert_one(t)` (which will upsert non-user triples as new).
   - After primary insert, apply auto-inverse writes:
     - If `(t.predicate, inverse)` appears in `AUTO_INVERSE_PAIRS`, construct `Triple { subject_id = t.object_id, object_id = t.subject_id, predicate = inverse, doc_ids = t.doc_ids.clone(), user_added = false, ... }` and `insert_one` it.
     - If `t.predicate` is in `SYMMETRIC_PREDICATES` AND `t.subject_id != t.object_id`, construct the symmetric triple `Triple { subject_id = t.object_id, object_id = t.subject_id, predicate = t.predicate.clone(), ... }` and `insert_one` it.
   Returns Vec of primary triple ids (auto-inverse ids are not returned to the caller — they are book-keeping).

8. `pub fn add_manual(&mut self, subject_id: String, predicate: String, object_id: String, doc_id: Option<String>) -> Result<String, String>` — called by `add_manual_triple` IPC. Builds a Triple with `user_added = true`, `doc_ids = doc_id.map(|d| vec![d]).unwrap_or_default()`, then calls `insert_one`. Applies auto-inverse + symmetric writes same as `upsert_from_doc` but with `user_added = true` on the inverse partner as well (per D-12, manual overrides propagate).

9. `pub fn delete(&mut self, triple_id: &str) -> bool` — remove from `triples` and all three indices. Also delete the auto-inverse / symmetric partner if one exists (look up by matching subject/predicate/object per AUTO_INVERSE_PAIRS + SYMMETRIC_PREDICATES). Returns true when at least one triple was removed.

10. `pub fn get_by_entity(&self, entity_id: &str) -> Vec<&Triple>` — return all triples where entity_id is subject OR object via `entity_touches` index. Sort by (predicate, object_id) for stable ordering.

11. `pub fn get_objects_for(&self, subject_id: &str, predicate: &str) -> Vec<&str>` — forward-index lookup: returns object_ids where (subject_id, predicate) matches. Empty when no match.

12. `pub fn get_subjects_for(&self, predicate: &str, object_id: &str) -> Vec<&str>` — reverse-index lookup: returns subject_ids where (predicate, object_id) matches.

13. `pub fn cleanup_doc(&mut self, doc_id: &str)` — called when a doc is re-indexed (D-25 discretion). Removes doc_id from every triple's `doc_ids`; if a triple ends up with empty doc_ids AND `user_added == false`, delete the triple entirely. User-added triples are preserved even when their last supporting doc disappears.

**Constants:** define a private helper `fn find_inverse_predicate(predicate: &str) -> Option<&'static str>` that scans `AUTO_INVERSE_PAIRS` and returns the paired inverse.
