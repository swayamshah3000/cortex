- Entity detail page (`/entity/:class/:value`) now shows a Relations section listing outgoing + incoming triples with clickable target-entity chips, predicate labels, doc-count badges, and per-row delete
- New `/ownership/:personId` page groups a person's assets into Property / Vehicle / Investment / Business / Financial / Other card grids; empty categories are hidden
- `DeleteTripleButton` (AlertDialog + `useDeleteTriple`) is shared by both surfaces — single source of truth for the confirm-delete UX

## Files Created/Modified

- `client/App.tsx` - Registered `<Route path="/ownership/:personId" element={<OwnershipPage />} />` next to the existing `/entity/:class/:value` route

## Decisions Made
- Kept `EmptyState` (with "Try syncing your folders" guidance) for the true-nothing case (no docs, no aliases beyond canonical, no co-occurring entities) rather than removing it entirely, since `EntityRelationsPanel` doesn't expose its relations count to the parent page — matches the plan's stated fallback design.
- `EntityRelationsPanel` treats an error from `useEntityRelations` as a silent `null` render (logged to console) so the page's own error/loading states remain the single source of truth, per plan spec.
