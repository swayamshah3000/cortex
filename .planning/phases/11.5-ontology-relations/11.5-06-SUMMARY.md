## Issues Encountered

The `rtk` shell hook intercepts bare `grep`/`find` commands and mangles argument parsing for pattern flags (e.g., `grep -c "a\|b"` failed with an unrelated error about `--max-len`). Worked around by using `rtk proxy grep ...` / `rtk proxy rg ...` for verification greps. No impact on delivered code — the underlying files and hooks are unaffected. This is an environment quirk, not a plan deviation.
