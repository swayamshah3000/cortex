<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Package registries → repo | npm/crates.io supplying potentially-malicious or typo-squat packages (T-06-SC) |
| HuggingFace LFS → repo | Bundled .onnx may be swapped (T-06-MODEL) |
| Web origin → asset protocol | Renderer requesting arbitrary local files via `asset://` URLs once enabled (T-06-AP) |
| OS shell ← opener plugin | `openPath`/`revealItemInDir` invoking arbitrary system handlers (T-06-OPN) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-SC | Tampering | npm/cargo installs (9 packages, all [ASSUMED] per RESEARCH §Package Legitimacy Audit) | mitigate | Blocking human-verify checkpoint (Task 1) BEFORE installs; cannot auto-advance |
| T-06-MODEL | Tampering | Bundled bert-base-NER.onnx from HuggingFace LFS | mitigate | Blocking human-verify checkpoint (Task 4) comparing shasum to LFS pointer |
| T-06-AP | Information Disclosure | Asset protocol scope `["**"]` permits renderer to fetch any local file | accept | Cortex indexes user-chosen folders; user has already trusted the app with these paths. react-markdown default escapes HTML so malicious .md files cannot inject `<img>` requests. Documented in tauri.conf.json comment if comments supported, else in this plan's SUMMARY. |
| T-06-OPN | Elevation of Privilege | `opener:allow-open-path` universal permission lets opener call shell handlers on any path | accept | Plan 04 wires openPath only from `doc.path` values that originate in watched-folder ingestion (user-trusted source). Documented as accepted tradeoff per RESEARCH §Pattern 3 Optional Hardening. |
| T-06-CSP | Information Disclosure / XSS | Current `csp: null` permits everything; tightening here narrows attack surface for Plan 05's markdown renderer | mitigate | Task 2 sets explicit CSP per RESEARCH §Pattern 2 — restricts script-src to 'self', frame-src/object-src/img-src to asset: only |
| T-06-PERM-DIAG | Spoofing / Repudiation | Without `dialog:allow-open` permission, the folder picker silently fails in production builds (different from dev) | mitigate | Permission added in Task 2; verified by Plan 04's WatchedPage smoke test in Tauri dev |
</threat_model>
