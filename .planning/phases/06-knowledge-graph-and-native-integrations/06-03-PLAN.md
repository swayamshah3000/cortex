Struct fields:
```
pub struct EntityStore {
    pub canonicals: HashMap<String, CanonicalEntity>,           // canonical_id -> entity
    pub alias_index: HashMap<(String, String), String>,         // (lowercase(value), entity_type) -> canonical_id
    pub doc_index: HashMap<String, std::collections::HashSet<String>>,  // canonical_id -> doc_ids
    pub canonical_embeddings: HashMap<String, Vec<f32>>,        // canonical_id -> embedding of canonical_name
}
```

Methods (all `pub` unless noted):
- `new() -> Self` — empty maps. Also impl Default.
- `rebuild_from_engine(&mut self, engine: &CortexEngine, embedder: &EmbeddingService) -> Result<(), AppError>` — O(N) scan of documents_384 collection per the edges.rs `build_edges` pattern (lines 47-87). Extract extracted_entities from each doc's metadata, for each entity call `find_or_create_canonical`, append doc_id to doc_index.
- `register_doc_entities(&mut self, doc_id: &str, entities: &mut [ExtractedEntity], embedder: &EmbeddingService) -> Result<Vec<String>, AppError>` — IMPORTANT: takes `&mut [ExtractedEntity]` so it can mutate each entity's `canonical_id` field in place. For each entity: call find_or_create_canonical with the entity's value+type, assign the returned canonical_id back onto `entity.canonical_id = Some(cid)`, append doc_id to doc_index[canonical_id]. Recompute canonical_name as most-frequent surface form. Return list of touched canonical_ids (parallel to the input slice — order preserved). Task 4 of this plan relies on the in-place mutation contract to write canonical_ids into the metadata that index_file persists.
- `find_or_create_canonical(&mut self, surface: &str, entity_type: &str, embedder: &EmbeddingService) -> Result<String, AppError>` — private. Algorithm:
  1. Exact-match lookup in alias_index for (lowercase(surface), entity_type) → return existing.
  2. Embedding-based search: embed surface, cosine vs every canonical_embedding of same type. If best score ≥ 0.85: append surface to canonical.aliases, register in alias_index, return existing canonical_id.
  3. New canonical: uuid::Uuid::new_v4(), insert into canonicals/alias_index/canonical_embeddings.
- `run_full_alias_merge(&mut self, embedder: &EmbeddingService) -> Result<(), AppError>` — for each entity_type, pairwise compare canonicals (O(n²) within type), merge if cosine ≥ 0.85. Per RESEARCH §Anti-Patterns this runs ONCE after backfill, not per-doc.
- `split_alias(&mut self, canonical_id: &str, alias_to_split: &str, embedder: &EmbeddingService, engine: &CortexEngine) -> Result<String, AppError>` — full mechanics from RESEARCH §Split-Alias Mechanics: create new uuid canonical, remove alias from old, rewrite affected doc metadata's extracted_entities[i].canonical_id from old→new via read-modify-write `collection.db.insert` upsert (mirror the `toggle_favorite` pattern in commands/documents.rs lines 186-234). Recompute document_count for both canonicals. Return new canonical_id.
- `rename_canonical(&mut self, canonical_id: &str, new_name: &str) -> Result<(), AppError>` — update canonicals[id].canonical_name only. No alias changes, no doc rewrites.
- `related_entities(&self, canonical_id: &str, min_co_occurrence: u32, limit: usize, engine: &CortexEngine) -> Result<Vec<RelatedEntity>, AppError>` — for each doc_id in doc_index[canonical_id], look up doc's extracted_entities (via engine + collection.db.get), count co-occurrences with other canonical_ids of the same doc. Apply filter ≥ min_co_occurrence, sort by count desc, truncate to limit, build `RelatedEntity { entity: EntitySummary{...}, co_occurrence_count }`.
- `get_by_type(&self, entity_type: Option<&str>) -> Vec<EntitySummary>` — filter canonicals by type (or all), map to EntitySummary, sort by document_count desc.
- `get_canonical(&self, canonical_id: &str) -> Option<CanonicalEntity>` — clone from canonicals map.

Lifecycle (per RESEARCH §Backfill Strategy lines 1226-1241):
1. Count docs needing backfill: scan documents_384, count entries where metadata.entities_version is missing OR < 2.
2. Emit initial event `{ processed: 0, total: N, status: "running", error: None }`.
3. If N == 0 emit `status: "complete"` and return.
4. Loop: for each candidate doc_id:
   - `tokio::task::spawn_blocking(move || backfill_one_doc(doc_id, engine_guard, ner_service, entity_store_guard, embedder)).await`
   - On Ok(true): processed += 1; if processed % 25 == 0 OR `last_emit.elapsed() >= 500ms`, emit progress event.
   - On Ok(false): break (no more docs).
   - On Err(e): emit `status: "error", error: Some(e.to_string())` and continue (per RESEARCH §Failure mode — don't abort entire backfill on one bad PDF).
5. After loop: acquire entity_store lock, call `store.run_full_alias_merge(&embedder)` (per D-06 a).
6. Final emit: `status: "complete"`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend IPC → entity commands | Untrusted args (entity_type strings, ids, alias strings, new_name strings) from renderer JS |
| Backfill task → engine | Concurrent reads/writes with the file watcher pipeline on the documents_384 collection |

| split_alias → engine | Mutates extracted_entities[].canonical_id on multiple docs; partial-failure mid-loop leaves the index inconsistent |
| index_file → EntityStore | Watcher / IPC ingest paths mutate the shared EntityStore on every new doc; embedder failures must not block indexing |
