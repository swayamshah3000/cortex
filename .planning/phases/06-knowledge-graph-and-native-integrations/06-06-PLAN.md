<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend IPC → get_entities_by_type | Untrusted entityType string arg from renderer; Plan 03 validates at boundary |
| /entities/:id route param → URL | canonicalId from the route is later passed to get_entity (Plan 07); this plan only emits Links, no IPC mutation |
| mockEntities → render | Mock data used in browser-dev only; never reaches production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-CHIP-LINK | Spoofing | EntityChip Link target uses `entity.canonicalId ?? encodeURIComponent(entity.value)` — pre-backfill, the value can be raw user text from a document | mitigate | `encodeURIComponent` neutralizes path-traversal characters; downstream EntityDetailPage (Plan 07) treats unknown ids as "no canonical found" state. No IPC call uses the raw value. |
| T-06-FILTER-INPUT | Tampering / Injection | EntityTypeFilterBar onSelect passes a type string into useEntitiesByType which calls IPC | mitigate | Filter pills are a fixed enum-like list ("all" + 6 types); user cannot type free-form values. IPC arg is strongly typed (Plan 03 entity_type: Option<String> matched against EntityStore canonical types). |
| T-06-MOCK-LEAK | Information Disclosure | Mock entity names accidentally referenced in production via tauriInvoke fallback | accept | tauriInvoke's fallback only fires when !isTauri(); production Tauri runtime never returns mock data. Same pattern as existing useDocument fallback. |
</threat_model>
