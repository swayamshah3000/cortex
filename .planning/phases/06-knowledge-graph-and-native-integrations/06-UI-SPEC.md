# Phase 6 — UI Design Contract

> Visual and interaction contract for `/entities` and `/entities/:id` routes, DocumentPage in-app preview + Open-in-OS actions, document-row context menu, native folder picker, and NER-backfill TopBar indicator. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase **extends** the existing Cortex design system (locked in Phases 1-5). It does NOT introduce a new visual language. Every new component reuses existing CSS-variable tokens defined in `client/global.css`, existing shadcn primitives in `client/components/ui/`, and existing Lucide icons. No new fonts, no new color palette, no new spacing scale.

| Property | Value |

## Typography

| Role | Size | Weight | Line Height | Cortex Class / Tailwind | Phase-6 Usage |
|------|------|--------|-------------|--------------------------|---------------|
| Body | 14px | 400 | 1.5 | `text-sm` | Document-list rows, alias rows, related-entity rows, metadata key/value pairs |
| Label | 12px | 500 | 1.4 | `text-xs font-medium` (badge-text class) | Type badges, kbd hints, sidebar section labels, "X documents" counts |
| Heading (section) | 18px | 600 | 1.3 | `section-header` class (`text-lg font-semibold`) | "Aliases", "Documents mentioning this", "Related entities" section titles |
| Display (page) | 24px | 600 | 1.2 | `page-title` class (`text-2xl font-semibold`) | `/entities` page title, `/entities/:id` canonical name in header |

**Phase-6 typography exceptions:**
- File paths in entity-detail page and `<TextPreview />` use `font-mono text-xs` (JetBrains Mono 12px) — matches existing `font-mono text-sm` on DocumentPage line 129 but stepped down to `text-xs` inside lists.
- Markdown rendered content uses `@tailwindcss/typography` plugin classes (`prose prose-invert max-w-none`) which set their own internal type scale. This is an opaque-but-locked third-party scale — accepted because (a) it is the conventional react-markdown styling, (b) it matches the existing dark theme via `prose-invert`, (c) it stays within px values that are multiples of 2 internally.
- Code blocks inside markdown use `font-mono text-sm` (overridden via prose customisation if Tailwind defaults clash).

**Forbidden in Phase 6:**
- No new font-size between 14px and 18px (15px / 16px / 17px banned for new components).
- No font-weight 300 (light) or 800+ outside the existing `.app-title` class.
- No italics for emphasis — emphasis comes from weight 500 or accent color, not italic.

**Contrast check (WCAG AA target 4.5:1 for normal text, 3:1 for large text and graphical elements):**
- Light mode: type-badge background uses `*/10` alpha over `bg-bg-secondary` (#FFFFFF) → very light tint. Foreground text uses the `-400` variant directly on this tint. Spot-checked: `text-purple-400` on `#FFFFFF`-tinted-purple passes 3:1 for graphical UI (icon + label inside badge). For body-text usage of entity type INSIDE a row (e.g. "John Smith / Person — 12 documents"), use `text-text-primary` for the name and `text-text-secondary` for the type-label, NOT the colored variant. This way the colored variant is reserved for the badge/icon only.
- Dark mode: `*/10` alpha over `bg-bg-secondary` (#1F1F2E) → slightly darker tint, foreground `-400` palette passes 4.5:1 against `bg-bg-secondary` in spot-check (Tailwind's `*-400` family is calibrated for dark backgrounds).

### Route 4: `/watched` — Watched Folders (MODIFIED)

**Phase-6 change (single line item):**

Remove the text-input "Add folder path" dialog entirely (WatchedPage.tsx lines 179-227 — `renderAddDialog()`).
