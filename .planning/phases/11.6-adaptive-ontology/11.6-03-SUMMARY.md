### Exact rule ordering in `normalize_entity` (as implemented)
1. `class == Some("Organization")` → `strip_corporate_suffixes(value)` (longest matching suffix from `CORPORATE_SUFFIXES`, case-insensitive, trims trailing `,`/`.`/space after cut).
2. `class == Some("Location")` → `last_hyphen_unit_segment(value)` (splits on `-` only; requires ≥ 2 hyphen-segments; tries last segment alone against `HYPHEN_UNIT_REGEX`, then falls back to last-two-segments joined with a space).
3. (Fallback, any class) whitespace collapse — `collapse_whitespace(value)` returned only if it differs from the original `value`.
4. Otherwise `None`.
5. Final guard: `None` if the candidate is empty or equals `entity.value` verbatim (idempotency + T-11.6-10 empty-string mitigation).

### Location of the three `extract_full` return-path call sites (post-edit line numbers)
- Path A (LLM disabled): `entity_normalizer::normalize_entities(&mut pass1_entities);` — inside the `if !self.llm_enabled...` block, ~line 106.

### Adjustments made to HYPHEN_UNIT_REGEX vs the plan
None — `HYPHEN_UNIT_REGEX = r"^[A-Za-z]{1,6}[- ]?\d+$"` matches the plan's Task 1 spec verbatim. The deviation was in the **split strategy** feeding into it: the plan's draft text explored splitting on `[- ]` (hyphen-or-space) but its own worked examples (`test_no_hyphen_returns_none` for "Riverside Complex P705") required hyphen-only splitting to avoid over-matching space-separated full names — the final plan text itself converges on this ("Update `last_hyphen_unit_segment` to split on '-' only"), which is exactly what was implemented.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Corporate-suffix chained-stripping over-stripped "Acme Corp Ltd"**
- **Found during:** Task 1 (writing `strip_corporate_suffixes`, running the plan's own unit tests)
- **Issue:** An initial repeated-strip-until-no-match implementation stripped " Ltd" from "Acme Corp Ltd" → "Acme Corp", then looped again and matched " Corp" → "Acme", which failed the plan's own `test_organization_ltd_stripped` (expects `Some("Acme Corp")`, single strip).
- **Fix:** Changed to a single pass that selects the LONGEST matching suffix (so "Foo Pvt Ltd" matches the combined " Pvt Ltd" entry directly, and "Acme Corp Ltd" matches only " Ltd" since no longer suffix in the list matches its tail).
- **Files modified:** `src-tauri/src/pipeline/entity_normalizer.rs`
- **Verification:** All 14 unit tests pass, including both `test_organization_ltd_stripped` and `test_organization_multiple_suffixes`.
- **Committed in:** `af21149` (Task 1 commit)

**2. [Rule 3 - Blocking, dependency substitution] once_cell not available — used std::sync::OnceLock**
- **Found during:** Task 1 (plan suggested `once_cell::sync::Lazy` or `std::sync::OnceLock`)
- **Issue:** `once_cell` is not a dependency in `Cargo.toml` (`regex = "1"` is present; `once_cell` is not). Adding it would require a new Cargo.toml edit and this crate's `rust-version = "1.77.2"` predates `std::sync::LazyLock` (stabilized 1.80).
- **Fix:** Used `std::sync::OnceLock` (stable since 1.70) with a `hyphen_unit_regex()` getter function — no new dependency, no MSRV bump. This was explicitly offered as an alternative in the plan text itself, not an unapproved substitution.
- **Files modified:** `src-tauri/src/pipeline/entity_normalizer.rs`
- **Verification:** Compiles cleanly; regex compiled exactly once via `OnceLock::get_or_init`.
- **Committed in:** `af21149` (Task 1 commit)

- **Found during:** Task 2 verification (`cargo test --lib graph::entity_store`)
