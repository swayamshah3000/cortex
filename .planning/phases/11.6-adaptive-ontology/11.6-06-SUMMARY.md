## Decisions Made

- **Result<T, String> over Result<T, AppError>:** Followed the plan's explicit signature spec. The codebase has both conventions (`commands/ai.rs` uses `String`, `commands/relations.rs` uses `AppError` directly since it derives `Serialize`). Kept new commands consistent with the plan rather than mixing conventions mid-module.
- **activity_log via `record_with_details`, not raw struct push:** The plan's pseudocode showed `al.push(ActivityEntry { ... })`, but the real type is `ActivityItem` behind a private `ActivityLog.items` field with a `record_with_details(action, subject, activity_type, document_id)` public API (matches `commands::spaces`/`documents`/`folders`). Used the real API — this is a Rule 1 auto-fix (the plan's pseudocode would not have compiled).
- **spawn_entity_backfill call site left unchanged:** Plans 04 (bootstrap worker) and 05 (backfill vocab lookup) have not executed in this worktree — `pipeline::backfill::spawn_entity_backfill` does not yet accept `ontology_store`/`auth` parameters. Updating the call site now would require inventing Plan 04/05's function signature ahead of time, which is out of this plan's scope (Rule 4 territory — deferred rather than guessed). `AppState.ontology_store` is constructed and available; Plan 04/05's executor updates the call site when it lands.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed activity_log emission to use the real `ActivityLog` API**
- **Found during:** Task 2 (commands/ontology.rs)
- **Issue:** Plan's interface pseudocode called `state.activity_log.lock()` then `al.push(crate::types::ActivityEntry { id, action, subject, timestamp, activity_type, document_id })` — but `ActivityEntry` doesn't exist (the real type is `ActivityItem`, constructed internally by `ActivityLog::record_with_details`); `items` is a private field with no public `push`.
- **Fix:** Added a `log_activity(state, action, subject, activity_type)` helper that calls `state.activity_log.lock()` then `log.record_with_details(action, subject, activity_type, None)`, matching `commands/spaces.rs`/`commands/documents.rs`/`commands/folders.rs` convention.
- **Files modified:** `src-tauri/src/commands/ontology.rs`
- **Verification:** `cargo check` compiles; all 7 mutation commands call `log_activity` with distinct action names.
- **Committed in:** `94c5c2f` (Task 2 commit)
