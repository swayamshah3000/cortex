## Accomplishments
- Added `SEED_PREDICATES` compile-time alias of the frozen Phase 11.5 `PREDICATE_VOCABULARY` (types.rs:783) — `PREDICATE_VOCABULARY` itself remains declared exactly once (types.rs:646) and is never mutated (D-03).

## Decisions Made
- `SEED_PREDICATES` is a genuine compile-time slice alias (`&[&str] = PREDICATE_VOCABULARY`), not a duplicated literal — verified by test `test_seed_predicates_aliases_baseline_vocabulary` asserting pointer-equal-content and length 21.
- TypeScript record type for a single ontology-store predicate entry is named `PredicateEntry`, not `Predicate`, because `Predicate` is already taken by the Phase 11.5 string-literal-union type (`(typeof PREDICATE_VOCABULARY)[number]`). This was called out explicitly in the plan and followed as specified.
- `canonical_short_name` backward compatibility verified with two new tests deserializing legacy JSON payloads that lack the `canonicalShortName` key entirely.

- **Fix:** Added `canonical_short_name: None,` to every affected struct literal (production code and test fixtures alike).
- **Files modified:** listed above.
