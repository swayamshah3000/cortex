<objective>
Ship the corpus-seed bootstrap: after the first BOOTSTRAP_MIN_DOCS (30) documents complete Pass 2 successfully, sample their titles + topics + tags + top-5 entities and call the active LLM once with BOOTSTRAP_PROMPT (D-01). Parse response into `BootstrapSeed`, validate + dedup against SEED_PREDICATES, apply via `OntologyStore::apply_bootstrap`, persist ontology.json. This is the "cortex wakes up smart for the user's domain" moment described in 11.6-CONTEXT.md §specifics.

Purpose: Without corpus-seed, cortex only has the 21 baseline Phase 11.5 predicates and Pass 3 must invent everything domain-specific on-the-fly (higher hallucination rate, no consistency across docs). Bootstrap is a single ~$0.005 Haiku call that primes the vocabulary before Pass 3 backfill runs at scale.

- `src-tauri/src/pipeline/ontology_bootstrap.rs` (new module)
- `BOOTSTRAP_PROMPT` const (~50-80 lines including few-shot example)

- `parse_bootstrap_json` and `validate_bootstrap` helper functions
- backfill.rs: pass2 success counter + trigger site

`pub const BOOTSTRAP_PROMPT: &str = r#"..."#`. Content (adapt as needed but keep the following structure):

```
You are helping cortex, a personal document intelligence tool, learn a user's domain-specific vocabulary from the first 30 documents in their library.

TASK:
Given titles + topics + tags + top-5 entities per document below, propose 15–30 domain-specific PREDICATES (relation types) and 5–15 ENTITY SUBCLASSES that would be useful for cortex to extract from this user's documents going forward.

CONSTRAINTS:
1. Output ONLY valid JSON — no markdown fences, no explanation.
2. Predicates use snake_case names ≤ 40 chars. Examples: `registered_to`, `insured_by`, `taxed_by`, `apartment_of`, `plot_in`, `sibling_of`, `mortgaged_to`, `sale_deed_of`, `tax_receipt_for`, `neighbor_of`.
3. Do NOT propose these — cortex already has them from Phase 11.5: owns, owned_by, purchased_from, sold_to, located_in, part_of, address_of, married_to, parent_of, child_of, member_of, employer_of, employee_of, partner_of, issued_by, dated, signed_by, uses_pan, uses_aadhaar, has_voter_id, mentioned_with.
4. For each predicate, provide `subject_class` and `object_class` when clear (both are one of: Person, Organization, Location, Date, Amount, Email, Phone, Identifier). Use null when ambiguous.
5. Entity subclasses refine one of the 8 classes. Examples: {class: "Location", subclass: "apartment"}, {class: "Location", subclass: "plot"}, {class: "Identifier", subclass: "vin"}, {class: "Person", subclass: "spouse"}. Provide an `example` (surface form) for each.
6. Prefer 3–10 char subclasses; avoid multi-word subclasses (`property_tax_receipt` → subclass:"tax_receipt").
7. Return an empty array when no domain-specific vocabulary is needed for a class of docs.

JSON SCHEMA:
{
  "predicates": [
    { "name": "snake_case_name", "description": "one-sentence use case", "subjectClass": "Person" | null, "objectClass": "Location" | null }
  ],
  "entitySubclasses": [
    { "class": "Location", "subclass": "apartment", "example": "Complex-Unit-204" }
  ]
}

FEW-SHOT EXAMPLE:
Documents:
- "Property Sale Deed 2024" [topic: property, tags: ownership, sale] entities: Alex Doe, Complex-Unit, Pune, 2024-04-11
- "Vehicle Registration Certificate" [topic: vehicle] entities: John Roe, Unit 204, MH12AB1234
```

`pub fn parse_bootstrap_json(raw: &str) -> Result<RawBootstrapOutput, AppError>`:
- Mirror Pass 3 pattern: two-attempt (direct + fence-stripped) parse.

`fn validate_bootstrap(raw: RawBootstrapOutput, seed_set: &HashSet<String>, existing_count: usize, now: &str, model_used: &str, sample_count: u32) -> BootstrapSeed`:
- Drop any predicate whose `name` is in `seed_set` (D-03).
- Drop any predicate with empty name, name > 40 chars, or name not matching `^[a-z][a-z0-9_]*$`.
- Drop duplicates (dedup by name within the batch).
- Cap at VOCABULARY_HARD_CAP - existing_count - SEED_PREDICATES.len() predicates so we never overflow the cap in a single bootstrap.
- Validate `class` in {Person, Organization, Location, Date, Amount, Email, Phone, Identifier}; drop otherwise.
- Validate `subclass` non-empty and ≤ 24 chars.
- Map into `BootstrapSeed { predicates, entity_subclasses, generated_at: now, sample_doc_count: sample_count, model_used }`.

`impl OntologyBootstrapper`:
- `pub fn new(auth, ontology_store, app_data_dir) -> Self`.
- `pub async fn bootstrap(&self, sample_docs: &[BootstrapSampleDoc]) -> Result<Option<BootstrapSeed>, AppError>`:
  1. Idempotency check: acquire ontology_store lock, check `bootstrap_completed()` — return Ok(None) if true.
  2. Provider check: `auth.get_active_provider()` → Ok(None) if None (D-04 mirror).
  3. Model selection: same convention as Pass 2 — configured extraction_model preferred; default per Pass2LlmRefiner::pick_model_default(provider).
  4. Format sample docs into a single user message:
     ```
     Documents (n={sample_docs.len()}):
     {for each doc: "- \"{title}\" [topic: {topic}, tags: {tags.join(",")}] entities: {top_entities.iter().map(|(c,v)| format!("{}:{}", c, v)).join(", ")}"}
     ```
  5. Construct `AIServiceRequest` with system=BOOTSTRAP_PROMPT, temperature=0.2 (D-02: modest creativity for domain inference, not full 0 like Pass 3), max_tokens=2048.
  6. Call `ai_request_with_retry(&self.auth, req, 3)`.
  7. Parse via `parse_bootstrap_json`.
  8. Validate via `validate_bootstrap` — obtain seed_set from `SEED_PREDICATES`.
  9. Acquire ontology_store lock; `apply_bootstrap(seed.clone(), &now_rfc3339)`; call `save(&self.app_data_dir)` — log warning on save error but do NOT propagate.
  10. Return Ok(Some(seed)).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11.6-12 | Denial of Service | Runaway predicate proposal from LLM | mitigate | `validate_bootstrap` caps at VOCABULARY_HARD_CAP - existing headroom; drops beyond that. |
| T-11.6-13 | Tampering | LLM proposing seed-name dups | mitigate | validate_bootstrap drops any predicate whose name is in seed_set. |
| T-11.6-14 | Tampering | Malformed predicate name (non-snake_case, empty, too long) | mitigate | Regex + length validation in validate_bootstrap. |
| T-11.6-15 | Denial of Service | Duplicate bootstrap on every backfill | mitigate | `OntologyStore.apply_bootstrap` is idempotent (Plan 02); counter + `bootstrap_completed()` gate prevents duplicate LLM calls. |
| T-11.6-16 | Elevation of Privilege | User-controlled sample docs steering LLM | accept | Single-user desktop app; user IS the operator of the LLM. No cross-user surface. |
