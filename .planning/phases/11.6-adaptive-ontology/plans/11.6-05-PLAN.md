<objective>
Convert Pass 3 from fixed 21-predicate vocabulary to runtime-supplied vocabulary drawn from OntologyStore.effective_predicate_names(). Allow Pass 3 to propose ≤ 3 new_predicates per document (D-05). Feed those proposals into OntologyStore.record_pending_predicate, which handles the ≥2-doc min-support gate (D-06) — promoted predicates then appear in the next Pass 3 call's vocabulary automatically.

Purpose: This is the adaptive-vocabulary contract. Without this, corpus-seed (Plan 04) sits inert in ontology.json and never influences extraction. With it, cortex's Pass 3 vocabulary grows from 21 → 21+corpus+adaptive predicates as the user's corpus is processed.

Output:
- `pipeline/pass3_relation_extractor.rs`: replace fixed EXTRACT_RELATIONS_PROMPT const with `build_extract_relations_prompt(vocab: &[String]) -> String` that injects the runtime vocab into the RULES §2 line. Retain few-shot examples untouched. `extract()` gains a `vocabulary: &[String]` parameter and calls the builder before constructing the AI request. `Pass3Output.new_predicates: Vec<Pass3NewPredicate>` added with #[serde(default)] and validated ≤ 3 per doc.
- `backfill.rs`: before each Pass 3 call, `let vocab = ontology_store.lock().await.effective_predicate_names();`; after Pass 3 succeeds, loop over `output.new_predicates` calling `ontology_store.record_pending_predicate(...)` with the doc_id.
</objective>

/// Build the full Pass 3 prompt with `vocab` injected into RULES §2.
/// Vocabulary is emitted as a comma-separated list in insertion order (matches
/// OntologyStore.effective_predicate_names() order — deterministic per D-05).
pub fn build_extract_relations_prompt(vocab: &[String]) -> String {
    let vocab_str = vocab.join(", ");

}

```rust
match two_pass.pass3().extract_full(&text, &title, &entities_vec, &vocabulary).await {
    Ok(Some((triples, new_predicates))) => {
        // Upsert triples to TripleStore (existing logic unchanged).
        let ts_clone = triple_store.clone();
        let doc_id_owned = doc_id.to_string();
        let triples_owned = triples;
        let upsert_result = tokio::task::spawn_blocking(move || -> Result<(), String> {
            let mut ts = ts_clone.blocking_lock();
            ts.cleanup_doc(&doc_id_owned);
            ts.upsert_from_doc(&doc_id_owned, triples_owned)?;
            Ok(())
        })
        .await
        .map_err(|e| crate::error::AppError::Internal(format!("Pass 3 triple upsert join error: {}", e)))?;

        match upsert_result {
            Ok(()) => {
                final_version = PASS3_TARGET_VERSION;

                // Phase 11.6: feed proposed new_predicates into OntologyStore pending queue.
                // Fire even when automatic_growth_enabled == false; the store guards promotion
                // with the min-support gate. Manual approval flow (Plan 06) exposes pending list to UI.
                if !new_predicates.is_empty() {
                    let os = ontology_store.clone();
                    let doc_id_owned = doc_id.to_string();
                    let now = chrono::Utc::now().to_rfc3339();
                    let np = new_predicates.clone();
                    let auto_growth_enabled = {
                        let g = os.lock().await;
                        g.automatic_growth_enabled()
                    };
                    if auto_growth_enabled {
                        let mut g = os.lock().await;
                        for p in &np {
                            let res = g.record_pending_predicate(
                                &p.name,
                                &p.description,
                                p.subject_class.clone(),
                                p.object_class.clone(),
                                &doc_id_owned,
                                &now,
                            );
                            if matches!(res, crate::types::PromoteResult::Promoted) {
                                eprintln!("[backfill] promoted new predicate '{}' after min-support met", p.name);
                            }
                        }
                    } else {
                        eprintln!("[backfill] {} new predicates proposed by Pass 3 but automatic ontology growth is disabled (Settings > Ontology)", np.len());
                    }
                }
            }
            Err(e) => {
                eprintln!("[backfill] Pass 3 triple upsert failed for doc {}: {} — leaving at 3.0", doc_id, e);
            }
        }
    }
    Ok(None) => {
        eprintln!("[backfill] Pass 3 skipped for doc {} (no provider or no pass2 entities)", doc_id);
    }
    Err(e) => {
        eprintln!("[backfill] Pass 3 error for doc {}: {} — leaving at 3.0", doc_id, e);
    }
}
```
