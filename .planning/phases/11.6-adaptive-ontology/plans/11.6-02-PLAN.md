```rust
pub struct SavedSearchStore { data: HashMap<String, SavedSearch>, ... }
impl SavedSearchStore {
    pub fn load(app_data_dir: &Path) -> Self {
        let path = app_data_dir.join("saved_searches.json");
        match std::fs::read_to_string(&path) {
            Ok(s) => serde_json::from_str(&s).unwrap_or_default(),
            Err(_) => Self::default(),
        }
    }
    pub fn save(&self, app_data_dir: &Path) -> std::io::Result<()> {
        let path = app_data_dir.join("saved_searches.json");
        let tmp  = path.with_extension("json.tmp");
        let json = serde_json::to_string_pretty(self)?;
        std::fs::write(&tmp, json)?;
        std::fs::rename(tmp, path)?;
        Ok(())
    }
}
```

```rust
    schema: OntologyStoreSchema,
```

`impl Default for OntologyStore` builds an empty schema (version = 1) plus `seed_set` populated from `SEED_PREDICATES`.

`OntologyStore::load(app_data_dir: &Path) -> Self`:
- path = `app_data_dir.join("ontology.json")`
- Match `fs::read_to_string(&path)`:
  - `Ok(s)` â†’ `serde_json::from_str::<OntologyStoreSchema>(&s).unwrap_or_default()` â†’ `Self::from_schema(schema)`.
  - `Err(_)` â†’ `Self::default()` (no file yet, or unreadable â€” silent-fail matches SavedSearchStore).
- Log `eprintln!("[ontology_store] loaded {} adaptive + {} pending + {} manual predicates from {}", ...)` after successful load.

`OntologyStore::from_schema(schema) -> Self`:
- Rebuild `name_to_source`: iterate SEED_PREDICATES â†’ insert Seed, then `schema.corpus_seed.predicates` â†’ Corpus, `schema.adaptive_predicates` â†’ Adaptive, `schema.manual_predicates` â†’ Manual. - Rebuild `class_to_subclasses`: iterate `schema.entity_subclasses` â†’ insert (class, subclass).
- seed_set = SEED_PREDICATES.iter().map(|s| s.to_string()).collect()

`OntologyStore::save(app_data_dir: &Path) -> io::Result<()>`:

`pub fn record_pending_predicate(&mut self, name: &str, description: &str, subject_class: Option<String>, object_class: Option<String>, doc_id: &str, now_rfc3339: &str) -> PromoteResult`:
- Reject empty name or names > 64 chars with `PromoteResult::CapExceeded` (defensive â€” LLM hallucination guard).
- If `name_to_source.contains_key(name)` â†’ return `PromoteResult::AlreadyPresent`.
- Look up in `schema.pending_predicates`:
  - Found â†’ increment `count`; if `count >= PENDING_PROMOTION_MIN_SUPPORT` AND `effective_predicate_names().len() < VOCABULARY_HARD_CAP` â†’ move entry from `pending_predicates` to `adaptive_predicates` with `source = Adaptive`, `promoted_at = Some(now_rfc3339)`; update `name_to_source`; return `PromoteResult::Promoted`.
  - Found and count still below min-support â†’ return `PromoteResult::StillPending { count }`.
  - Found but cap exceeded â†’ keep in pending, return `PromoteResult::CapExceeded`.
- Not found â†’ push new Predicate to `pending_predicates` with `source = Adaptive`, `count = 1`, `first_seen_doc_id = Some(doc_id.to_string())`, `first_seen_at = Some(now_rfc3339.to_string())`, `promoted_at = None`; return `PromoteResult::StillPending { count: 1 }`.

`pub fn apply_bootstrap(&mut self, seed: BootstrapSeed, now_rfc3339: &str)`:
- Idempotency: if `bootstrap_completed_at.is_some()`, return early (no-op). D-02: bootstrap runs once per install.
- Store `schema.corpus_seed = Some(seed.clone())`.
- For each predicate in seed.predicates:
  - Skip if `seed_set.contains(&p.name)` (D-03: seed always wins).
  - Insert into `name_to_source` with source=Corpus.
- For each entity_subclass in seed.entity_subclasses:
  - Append to `schema.entity_subclasses` with source=Corpus (dedup by (class, subclass) pair).
  - Update `class_to_subclasses`.
- Set `schema.bootstrap_completed_at = Some(now_rfc3339.to_string())`.

`pub fn register_manual_predicate(&mut self, name: String, description: String, subject_class: Option<String>, object_class: Option<String>, now_rfc3339: &str) -> PromoteResult`:
- Validate: name non-empty, len â‰¤ 64, matches `^[a-z][a-z0-9_]*$` (snake_case). Reject with `PromoteResult::CapExceeded` on validation fail.
- If `name_to_source.contains_key(&name)` â†’ `PromoteResult::AlreadyPresent`.
- If `effective_predicate_names().len() >= VOCABULARY_HARD_CAP` â†’ `PromoteResult::CapExceeded`.
- Push to `schema.manual_predicates` with source=Manual, count=0, first_seen_at=Some(now); update index; return `Promoted`.

`pub fn rename_predicate(&mut self, old: &str, new: &str, now_rfc3339: &str) -> Result<(), AppError>`:
- Reject rename of seed predicates (D-03): if `seed_set.contains(old)` return `AppError::Invalid("cannot rename seed predicate")`.
- Validate `new` per manual predicate rules.
- Find `old` in corpus_seed.predicates OR adaptive_predicates OR manual_predicates OR pending_predicates; mutate in place (name = new.to_string(), promoted_at = Some(now)); update index (remove old key, insert new).
- Not found â†’ `AppError::NotFound("predicate: {old}")`.
- Note: this method mutates only vocabulary. Rewriting existing triples in TripleStore is the caller's responsibility (Plan 06 apply_consolidation instruction return type).

`pub fn merge_predicates(&mut self, from: Vec<String>, into: String, now_rfc3339: &str) -> Result<(), AppError>`:
- Reject merging seed predicates (D-03).
- Ensure `into` exists in the effective vocabulary; if not, return NotFound.
- For each name in `from`: remove from adaptive/manual/pending/corpus_seed lists; remove from index. Add `into.count += from_entry.count` where applicable.
- If any name in `from` equals `into`, skip that entry (self-merge no-op).
- Returns Ok(()) on success.

`pub fn reset_to_seed(&mut self, now_rfc3339: &str)`:
- Wipe `corpus_seed`, `adaptive_predicates`, `pending_predicates`, `manual_predicates`, `entity_subclasses`, `pending_consolidation`.
- Preserve `automatic_growth_enabled` (user preference is user-owned, not derived from corpus).
- Set `bootstrap_completed_at = None` so next backfill batch of 30 docs re-triggers bootstrap.
- Rebuild indices from empty state + seed_set.

`pub fn set_pending_consolidation(&mut self, pc: Option<PendingConsolidation>)`:
- Store as-is.

`pub fn pending_consolidation(&self) -> Option<PendingConsolidation>`:
- Return clone.

`pub fn apply_consolidation(&mut self, suggestion_id: &str, now_rfc3339: &str) -> Result<Vec<TripleRewriteInstruction>, AppError>`:
- Define `pub struct TripleRewriteInstruction { pub kind: TripleRewriteKind, pub from: Vec<String>, pub to: String }` and `pub enum TripleRewriteKind { Rename, Merge, Split }`.
- Locate suggestion by id in `schema.pending_consolidation.suggestions`; return NotFound if missing.
- Apply the mutation to the ontology:
  - `ConsolidationKind::Rename { from, to }` â†’ call `self.rename_predicate(&from, &to, now)`; return one `TripleRewriteInstruction { kind: Rename, from: vec![from], to }`.
  - `ConsolidationKind::Merge { from, into }` â†’ call `self.merge_predicates(from.clone(), into.clone(), now)`; return one `TripleRewriteInstruction { kind: Merge, from, to: into }`.
  - `ConsolidationKind::Split { from, into }` â†’ for each name in `into`, `self.register_manual_predicate(name.clone(), format!("Split from {from}"), None, None, now)`; remove `from` predicate from adaptive/manual/pending (not seed, error if seed); return `TripleRewriteInstruction { kind: Split, from: vec![from], to: String::new() }` with `to = ""` (caller handles split by deferring to user â€” Phase 11.6 does not auto-rewrite splits, only records them).
- Remove suggestion from pending consolidation. If suggestions becomes empty, set `pending_consolidation = None`.
- Update `schema.last_consolidation_at = Some(now_rfc3339.to_string())`.
- Return the TripleRewriteInstruction Vec.
