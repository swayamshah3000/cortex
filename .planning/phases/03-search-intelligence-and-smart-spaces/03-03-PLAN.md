  **graph/edges.rs — Document edge construction (INTL-03):**

  Do NOT depend on ruvector-graph for v1. The ruvector-graph crate is a full property graph database with Cypher parser, ACID transactions, and distributed support — massive overkill for storing document relationships. Instead, use a simple in-memory adjacency list:

  - `struct DocumentEdge { source: String, target: String, weight: f64, edge_types: Vec<String> }`
  - `struct DocumentGraph { edges: HashMap<String, Vec<DocumentEdge>> }`
    - Adjacency list: doc_id -> list of edges from that doc.
  - `DocumentGraph::new() -> Self`
  - `DocumentGraph::build_edges(&mut self, engine: &CortexEngine, space_manager: &SpaceManager) -> Result<(), AppError>`
    1. Read ALL vectors from documents_384 collection.
    2. For each pair of documents (or limit to top-N nearest neighbors to avoid O(n^2)):
       - Compute cosine similarity.
       - If similarity > 0.7: add "content_similarity" edge with weight = similarity.
    3. For documents sharing a space: add "shared_space" edge with weight = 0.5.
    4. For documents sharing tags: add "shared_tags" edge with weight = 0.3 * (shared_tag_count / max_tags).
    5. For documents sharing entities (same person name, same organization): add "shared_entity" edge with weight = 0.4.
    6. Merge parallel edges between same pair: combine weights (max, not sum, to avoid over-counting).
    - **Scalability note:** For the pair-wise similarity check, do NOT compare every document to every other document. Instead, for each document, use the HNSW search to find top-10 nearest neighbors and only create edges for those. This is O(n * log(n)) instead of O(n^2).
  - `DocumentGraph::get_neighbors(&self, doc_id: &str, limit: usize) -> Vec<&DocumentEdge>`
    - Return edges sorted by weight descending, capped at limit.
  - `DocumentGraph::build_space_graph(&self, space_manager: &SpaceManager) -> SpaceGraph`
    - Create one SpaceGraphNode per space (id, name, color, document_count).
    - Create SpaceGraphEdge between spaces that share documents or have high inter-space similarity.
    - Edge weight = number of shared documents + average content_similarity between spaces.

  **graph/related.rs — Related documents retrieval (SPAC-05):**
  - `fn get_related_impl(doc_id: &str, limit: usize, graph: &DocumentGraph, engine: &CortexEngine) -> Result<Vec<Document>, AppError>`
    1. Get neighbors from graph.
    2. For each neighbor doc_id, look up metadata in RuVector collection.
    3. Build Document structs from metadata.
    4. Return sorted by edge weight descending, capped at limit.
