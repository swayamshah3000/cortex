  **SPAC-07 — Domain expansion bootstrapping:**
  Do NOT add ruvector-domain-expansion as a dependency. The domain expansion crate is a full meta-learning system (Thompson Sampling, policy kernels, Rust synthesis domain, etc.) designed for cross-domain transfer in AGI research. It's overkill for Cortex v1's needs.

  Instead, implement a lightweight version in spaces/manager.rs:
  - When recluster detects a new cluster that wasn't in the previous space set:
    1. Find the closest existing space by centroid similarity.
    2. If similarity > 0.6, the new space "bootstraps" from the existing one:
       - Copy relevant naming hints (e.g., if closest space is "Work" and new cluster has similar entity types, name might be "Work - Projects").
       - Copy tag patterns.
    3. If similarity < 0.6, the new space starts fresh (no bootstrap).
  - This is the v1 "domain expansion" — when new document clusters appear, they learn naming conventions from nearby existing spaces.
  - Add a `previous_spaces: Vec<SpaceData>` field to SpaceManager that stores the last clustering result for comparison.

  **SRCH-05 — Search-as-you-type (150ms debounce):**
  The debounce happens on the FRONTEND side (React Query or custom hook with 150ms setTimeout). The backend just needs to be fast enough for partial queries:
  - In search/query.rs, add an early-return optimization: if query text is less than 3 characters, return empty results immediately (avoid embedding very short strings).
  - This is not a backend change per se — it's a frontend concern. But document it in SUMMARY so Phase 4 frontend knows to add the 150ms debounce.
  - Add a min_query_length parameter (default 3) to search_documents_impl.

  **FWAT-05 — Background indexing progress events:**
  - Already implemented in watcher/worker.rs (the IndexProgress struct and emit calls exist from Phase 2).
  - Verify that the existing code emits "index-progress" events with status "indexing", "indexed", "skipped", "error".

  - In commands/folders.rs trigger_scan, ensure it actually walks the folder and indexes all files (not just the stub).

  - Already implemented in pipeline/indexer.rs (content hash comparison in index_file).
