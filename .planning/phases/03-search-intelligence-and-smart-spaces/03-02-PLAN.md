  **spaces/clustering.rs — Document clustering (SPAC-01, SPAC-02, SPAC-04):**

  Implement a simple but effective clustering algorithm. Do NOT depend on ruvector-gnn for the initial implementation — ruvector-gnn is a training framework (Adam optimizer, replay buffers, EWC) not a plug-and-play clustering library. Instead, implement k-means with cosine similarity directly:

  - `struct ClusterResult { clusters: Vec<Cluster> }`
  - `struct Cluster { id: String, doc_ids: Vec<String>, centroid: Vec<f32> }`
  - `fn cluster_documents(vectors: Vec<(String, Vec<f32>)>, k: usize) -> ClusterResult`
    1. If fewer than k documents, put each in its own cluster.
    2. Initialize k centroids using k-means++ (pick first randomly, subsequent by max distance).
    3. Iterate (max 20 iterations):
       a. Assign each document to nearest centroid by cosine similarity.
       b. Recompute centroids as mean of assigned vectors.
       c. If no assignments changed, break early.
    4. Return clusters with their centroid vectors and member doc IDs.
  - `fn auto_detect_k(n_documents: usize) -> usize`
    - Heuristic: sqrt(n/2), clamped to [2, 20].
    - This avoids needing the user to specify cluster count.
  - `fn cosine_similarity(a: &[f32], b: &[f32]) -> f32` — local helper (same as ruvector-gnn search.rs).
  - `fn normalize(v: &mut [f32])` — L2 normalize in place.

  Why not ruvector-gnn: The GNN crate provides deep learning primitives (layers, optimizers, loss functions) meant for training graph neural networks. For document clustering in Cortex v1, k-means with cosine similarity is the right tool — fast, deterministic, no training needed. Future versions can upgrade to ruvector-gnn's SubGraph extraction for more nuanced clustering.

  **spaces/naming.rs — Rule-based space naming (SPAC-03):**
  - `fn name_space(doc_metadata: &[HashMap<String, serde_json::Value>]) -> (String, String, String)`
    - Returns (name, icon, color).
    - Algorithm:
      1. Collect all entity types across member documents: count occurrences of each entity_type (date, amount, person, organization).
      2. Collect all doc_types: count pdf, docx, xlsx, etc.
      3. Collect path segments: extract parent directory names, count occurrences.
      4. Heuristic naming rules:
         - If dominant entity is "amount" and doc_type is "pdf" -> "Invoices & Receipts" (icon: "Receipt", color: "#10B981")
         - If dominant entity is "date" and path contains "tax" or "property" -> "Property" (icon: "Home", color: "#6366F1")
         - If dominant entity is "person" or "organization" -> "Contacts & Correspondence" (icon: "Users", color: "#F59E0B")
         - If doc_type is predominantly "xlsx" or "csv" -> "Spreadsheets & Data" (icon: "Table", color: "#3B82F6")
         - If path contains "work" or "project" -> "Work" (icon: "Briefcase", color: "#8B5CF6")
         - If path contains "medical" or "health" -> "Medical" (icon: "Heart", color: "#EF4444")
         - If path contains "kid" or "school" or "education" -> "Kids & Education" (icon: "GraduationCap", color: "#F97316")
         - Default: "Space {N}" (icon: "Folder", color: "#6B7280")
      5. These rules are intentionally simple for v1. LLM-based naming (Ollama) is deferred to ASPAC-02 in v2.

  **spaces/manager.rs — Space state management (SPAC-06):**
  - `struct SpaceManager { spaces: HashMap<String, SpaceData>, doc_to_space: HashMap<String, Vec<String>> }`
  - `struct SpaceData { space: Space, centroid: Vec<f32>, doc_ids: Vec<String> }`
  - `SpaceManager::new() -> Self`
  - `SpaceManager::recluster(&mut self, engine: &CortexEngine) -> Result<Vec<Space>, AppError>`
    1. Read ALL vectors from documents_384 collection (iterate keys, get each entry).
    2. Collect (doc_id, vector) pairs.
    3. Auto-detect k.
    4. Run cluster_documents.
    5. For each cluster, read member doc metadata and call name_space.
    6. Build Space structs with document_count, last_updated, sample_files.
    7. Update internal state (spaces map, doc_to_space map).
    8. Return the new spaces.
  - `SpaceManager::get_spaces(&self) -> Vec<Space>` — return all spaces.
  - `SpaceManager::get_space_documents(&self, space_id: &str) -> Vec<String>` — return doc IDs in a space.
  - `SpaceManager::move_document(&mut self, doc_id: &str, target_space_id: &str) -> Result<(), AppError>`
    - Remove doc from current space(s).
    - Add doc to target space.
    - Update document_count for both spaces.
    - Do NOT trigger re-cluster (SPAC-06 requirement).
    - Update centroid vectors for affected spaces (recalculate mean).

  - Test auto_detect_k with various document counts.
  - Test name_space with invoice-like metadata -> returns "Invoices & Receipts".
  - Test SpaceManager::move_document moves without re-clustering.
