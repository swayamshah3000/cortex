  **intelligence/sona_bridge.rs — SONA self-learning integration (INTL-01):**
  - `struct SearchLearner { engine: SonaEngine }`
  - `SearchLearner::new(embedding_dim: usize) -> Self`
    - Create SonaEngine with hidden_dim = embedding_dim (384 for local ONNX).
  - `SearchLearner::record_search(&self, query_embedding: &[f32], result_scores: &[f32]) -> Result<(), AppError>`
    1. Begin trajectory with query_embedding.
    2. For each result score, add a step with the score as reward.
    3. End trajectory with overall quality = mean of result scores.
    4. The SONA engine processes this internally via its learning loops.
  - `SearchLearner::record_click(&self, query_embedding: &[f32], clicked_doc_embedding: &[f32], position: usize)`
    - Creates a learning signal weighted by position (higher weight for clicking lower-ranked results — the system should learn from corrections).
    - Begin trajectory, add step with clicked embedding and reward = 1.0 / (1 + position), end with quality.
  - `SearchLearner::apply_boost(&self, query_embedding: &[f32]) -> Vec<f32>`
    - Apply micro-LoRA to the query embedding to get a boosted version.
    - If SONA hasn't learned anything yet, returns the original embedding unchanged.

  **intelligence/reranker.rs — Attention re-ranking (SRCH-06):**
  - `fn rerank_results(query_vec: &[f32], results: &mut Vec<SearchResult>, result_vecs: &[Vec<f32>])`
    1. Use ruvector-attention's ScaledDotProductAttention with dim = 384.
    2. Compute attention weights: query is the query vector, keys and values are the result vectors.
    3. The attention output gives a weighted combination, but we want the attention WEIGHTS (not the output).
    4. Actually, for re-ranking we want the raw attention scores before softmax, or use the attention weights directly as ranking scores.
    5. Simple approach: compute attention.compute(query, keys, values) to get weights. Then blend: final_score = 0.7 * original_cosine_score + 0.3 * attention_weight.
    6. Re-sort results by final_score descending.
  - If ruvector-attention's API doesn't directly expose weights, fall back to manual scaled dot-product:
    ```rust
    fn attention_score(query: &[f32], key: &[f32]) -> f32 {
        let dot: f32 = query.iter().zip(key).map(|(a, b)| a * b).sum();
        dot / (query.len() as f32).sqrt()
    }
    ```
    Then softmax over all scores and blend with original.

  **intelligence/analytics.rs — Search analytics (INTL-02):**
  - `struct SearchTracker { queries: Vec<QueryRecord>, total_searches: u32 }`
  - `struct QueryRecord { query: String, result_count: usize, timestamp: String, clicked_index: Option<usize> }`
  - `SearchTracker::new() -> Self`
  - `SearchTracker::record_query(&mut self, query: &str, result_count: usize)`
    - Push QueryRecord, increment total_searches.
    - Keep max 1000 records (ring buffer — remove oldest when full).
  - `SearchTracker::record_click(&mut self, query_index: usize, clicked_result_index: usize)`
    - Update the QueryRecord at query_index with clicked_index.
  - `SearchTracker::get_analytics(&self) -> SearchAnalytics`
    - total_searches from counter.
    - top_queries: group by query string, sort by frequency, take top 10.
    - avg_results_per_query: mean of result_count across all records.
