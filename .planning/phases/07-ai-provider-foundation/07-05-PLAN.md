## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (paste field) → useSaveSetupToken / useConnectProvider | Token / API key entered by user; passed through tauriInvoke to backend |
| Network: Ollama dynamic model fetch (browser fetch) | UI fetches `${baseUrl}/api/tags` from the user's local Ollama server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-16 | Information Disclosure | Setup token logged in browser console on paste | mitigate | Password input `type="password"` prevents shoulder-surfing; token value lives only in component state and is sent to Tauri IPC. NO `console.log` of token value. Verified by grep for `console.log` near `useSaveSetupToken` call site. |
| T-07-17 | Spoofing | User pastes a fake `sk-ant-oat01-...` prefix string | mitigate | Backend `validate_anthropic_token` makes a real API call — fake tokens get HTTP 401 and rejected before storage. UI shows the mapped error toast. |
| T-07-18 | Tampering | Malicious Ollama base URL pointing at non-Ollama service | accept | Single-user desktop app; user enters their own URL. Backend's `ollama_chat` makes a POST to `${base_url}/api/chat` — a hostile URL gets a parse error from the response and surfaces via toast. SSRF risk mitigated by reqwest default (no redirect following). |
| T-07-19 | Information Disclosure | Ollama dynamic model fetch hits an arbitrary URL | accept | The browser `fetch` happens inside the Tauri webview; CSP must allow it (Cortex Phase 1 set CSP). If users enter a non-localhost URL, that's their explicit choice. |
| T-07-SC | Tampering | No new npm install in this plan | mitigate | All imports resolve to already-installed packages (sonner, lucide-react, react-query, radix-ui). No package install task; legitimacy audit not required. |
</threat_model>
