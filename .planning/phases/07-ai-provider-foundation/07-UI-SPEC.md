## Spacing Scale

All spacing values are multiples of 4px. Tokens reuse the established Cortex 4px base grid.

| Token | Value | Usage in Phase 7 |
|-------|-------|------------------|
| xs | 4px | Icon-to-text gaps (e.g. provider name + status badge, button icon + label) |
| sm | 8px | Intra-card element spacing (radio + label, badge + text) |
| md | 16px | Card internal padding (horizontal and vertical), inline form field spacing |
| lg | 24px | Card-to-card gap in stacked provider list; onboarding step vertical rhythm |
| xl | 32px | Section gap (Embedding Model section bottom → divider → AI Providers section top) |
| 2xl | 48px | Onboarding container vertical padding (top/bottom of wizard content area) |
| 3xl | 64px | Not used in Phase 7 surfaces |

**Exceptions:**
- Onboarding 2×2 provider grid: 16px gap between cards (already standard — `gap-4`).
- Banner: 16px horizontal AND vertical padding (`px-4 py-4`) — compact single-line bar uses the standard `md` token.

> Note: 12px was removed from this contract. All row and banner padding uses 16px (`py-4`). The value 12px is not in the standard set {4, 8, 16, 24, 32, 48, 64} and is not used in Phase 7.

## Typography

Reuses the established Cortex type scale from `client/global.css`. Phase 7 uses exactly **3 sizes** and **2 weights**.

| Role | Size | Weight | Line Height | Font | Usage in Phase 7 |
|------|------|--------|-------------|------|------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Inter | Provider card descriptions, instruction text, badge labels, form helper text, status badge copy |
| Label / Heading | 18px (text-lg) | 600 (semibold) | 1.3 | Inter | "AI Providers" section header (uses `.section-header` token), onboarding step h2 (uses `.section-title`), provider name in card header, form field labels |
| Display | 24px (text-2xl) | 600 (semibold) | 1.2 | Inter | Onboarding step h1 (uses `.page-title` token) — Welcome and Connect AI step titles only |

**Size consolidation rationale:**
- 13px mono code block → consolidated to **14px** (matches Ollama URL mono input, consistent mono size throughout).
- 12px badge text → consolidated to **14px** (text-sm) — badges remain fully readable; the existing `.badge-text` token (`text-xs`) is overridden in Phase 7 components to use `text-sm` for consistency.
- Weight 500 (medium) dropped — **400 (normal) for body**, **600 (semibold) for all labels, headings, and display**. Matches existing `.section-header` / `.page-title` semibold tokens in globals.css.

**Code / monospace:**
- Setup-token code block: JetBrains Mono, **14px (text-sm)**, weight 400 — used in the Anthropic card connect form for the `claude setup-token` command snippet.
- Ollama base URL input: JetBrains Mono, 14px, weight 400 — monospace treatment signals this is a URL/path.
- Both use existing `font-family: "JetBrains Mono", monospace` via the `code` base rule in globals.css.

**Status badges (Connected / Not connected / Validating… / Failed):**
- Size: 14px (text-sm), weight 600 (semibold) — uses `text-sm font-semibold` directly. Overrides the global `.badge-text` token (`text-xs font-medium`) for Phase 7 provider status badges to stay within the 2-weight constraint.

## Color

Reuses all Cortex design tokens from `client/global.css`. No new color values are introduced.

| Role | Dark Mode Token | Light Mode Token | Usage in Phase 7 |
|------|----------------|-----------------|------------------|
| Dominant (60%) | `--bg-primary` (hsl 240 9% 8%) | `--bg-primary` (hsl 0 0% 98%) | Page background, onboarding wizard background |
| Secondary (30%) | `--bg-secondary` (hsl 240 9% 14%) | `--bg-secondary` (hsl 0 0% 100%) | Provider cards, expanded card form areas, banner background |
| Accent (10%) | `--accent-primary` (hsl 270 72% 60% dark / 270 91% 42% light) | same token | See reserved-for list below |
| Destructive | `--error` (hsl 0 72% 51% dark / 0 84% 60% light) | same token | Disconnect button text/border; inline red status badge on failed provider card |

**Accent reserved for (exclusive list):**
1. Active-provider radio button fill (selected state)
2. "Connect" CTA button background in onboarding provider cards and Settings AI tab
3. "Continue" button in onboarding Connect AI step (after any provider connects)
4. Step indicator pill — active step (wide pill at `w-8 bg-accent-primary`, existing StepIndicator pattern)
5. Focus ring on all form inputs (existing `.input-base` focus ring)
6. Provider card border highlight on hover (`hover:border-accent-primary/50`)
7. "Saved" flash state on Save button (text color `text-success` — NOT accent)

**Status semantics (distinct from accent):**
- `--success` (hsl 160 84% 54% dark): "Connected" badge background tint, "Saved" green flash button state
- `--error` (hsl 0 72% 51% dark): "Not connected" / failed validation inline badge; Disconnect button; error toast Sonner uses its own red by default
- `--warning` (hsl 40 96% 56%): Provider-missing nudge banner icon (AlertTriangle) — softer than error since it's a nudge, not a failure

**Token usage pattern (ALWAYS use tokens, never hardcode):**
- `text-accent-primary`, `bg-accent-primary`, `border-accent-primary` — Tailwind CSS custom properties resolved from `--color-accent-primary`
- `text-success`, `text-error`, `bg-error/10`, `bg-accent-primary/10` — opacity modifiers on tokens
- `bg-red-400/10`, `text-red-400` pattern seen in BackfillIndicator is acceptable for error pill; prefer `bg-error/10 text-error` for consistency in new Phase 7 components

## Interaction Contracts

### Validation UX (D-08, D-10)

**Save / Connect button lifecycle:**
1. Default: `"Connect"` or `"Save"` — `btn-primary` styling, enabled
2. On click: button text changes to `"Validating…"`, spinner `<Loader2 size={14} className="animate-spin" />` prepended, button disabled
3. On success: button text flashes to `"Saved"` with `text-success` color for 1200ms, then reverts to management state (Disconnect button appears, Connect button gone)
4. On error: button returns to `"Connect"` / `"Save"` state, enabled; Sonner `toast.error()` fires with provider-specific error message; provider card shows inline red status badge

**Spinner implementation:** inline within button using `{isPending ? <Loader2 size={14} className="animate-spin" /> : null}` — no full-card overlay (D-10 explicit).

### Provider Card Expand / Collapse

- Chevron icon: `ChevronDown` from Lucide, `transition-transform duration-200`, rotates `rotate-180` when open
- Expansion: CSS max-height transition OR shadcn `Collapsible` (already in shadcn registry)
- Only one card can be in expanded-connect state at a time (clicking Connect on a second card collapses the first)
- Connected cards can all be expanded simultaneously (for model/disconnect management)

### Active Provider Radio (D-17)

- Uses `RadioGroup` + `RadioGroupItem` (already imported in SettingsPage)
- Radio disabled when `!status.authenticated` — `<RadioGroupItem disabled={!status.authenticated} />`

- Visual: disabled radio has `opacity-50 cursor-not-allowed`

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Status badges | `role="status"` aria attribute on Connected/Not-connected badge |
| Banner | `role="alert"` with `aria-live="polite"` — appears on mount when conditions met |
| Validation spinner | `aria-busy="true"` on Connect button during validation; `aria-label="Validating connection"` |
| Provider cards | `aria-expanded` on chevron button; `aria-controls` pointing to form panel id |
| Radio group | `aria-disabled="true"` on radio items for unconnected providers |
| Password inputs | `autoComplete="off"` on all API key / token inputs (existing pattern from SettingsPage) |
| Error messages | `aria-describedby` linking input to inline error text when validation fails |
| Copy button (Anthropic card) | `aria-label="Copy command"` — icon-only button requires explicit label for screen readers |

**Accessibility constraints (non-spacing):**
- **Minimum touch target:** 44×44px (WCAG 2.5.5 AAA / Apple HIG). Applies to all interactive controls: Connect/Disconnect buttons, radio hit areas, chevron expand buttons, banner dismiss button. This is a minimum size requirement, not a spacing token — do not add 44px to the spacing scale. Implement via `min-h-[44px] min-w-[44px]` on each control independently.
