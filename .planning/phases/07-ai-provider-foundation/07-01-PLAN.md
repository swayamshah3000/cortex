#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "kebab-case")]  // CRITICAL: "api-key" on the wire, not "api_key"
pub enum AuthMethod { OAuth, ApiKey, None }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProviderCredential {
    pub provider: String,
    pub method: AuthMethod,
    pub api_key: Option<String>,
    pub oauth_token: Option<String>,
    pub display_name: Option<String>,
    pub model: Option<String>,
    pub base_url: Option<String>,
}

#[derive(Clone)]
pub struct AuthState {
    pub(crate) store_path: PathBuf,
    pub(crate) store: Arc<Mutex<CredentialStore>>,
}

impl AuthState {
    pub fn new(app_data_dir: &PathBuf) -> Self;
    pub fn store_api_key(&self, provider: &str, api_key: &str, model: Option<&str>) -> Result<(), String>;
    pub fn store_oauth_token(&self, provider: &str, token: &str, display_name: Option<&str>, model: Option<&str>) -> Result<(), String>;
    pub fn store_ollama_config(&self, base_url: &str, model: Option<&str>) -> Result<(), String>;
    pub fn get_credential(&self, provider: &str) -> Result<Option<ProviderCredential>, String>;
    pub fn get_active_credential(&self) -> Result<Option<ProviderCredential>, String>;
    pub fn get_active_provider(&self) -> Result<Option<String>, String>;
    pub fn set_active_provider(&self, provider: &str) -> Result<(), String>;
    pub fn remove_credential(&self, provider: &str) -> Result<(), String>;
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → Tauri IPC | Frontend cannot read raw credentials; only `ProviderAuthStatus` (no token) crosses |
| Tauri process → disk | `credentials.json` is plaintext in `app_data_dir`; OS file permissions are the only protection (v1.1 accepted per OOS table) |
| Tauri process → Anthropic API | Setup-token validation call goes over HTTPS to api.anthropic.com |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | `ProviderAuthStatus` over IPC | mitigate | Struct intentionally omits `api_key`/`oauth_token` fields — only `authenticated: bool`, `method: String`, `display_name`, `model`, `is_active` cross the wire. Verified via grep that no IPC return type includes raw token fields. |
| T-07-02 | Information Disclosure | `credentials.json` on disk | accept | Plaintext JSON is v1.1 OOS exception (REQUIREMENTS.md: "macOS Keychain credential storage in v1.1 — Plaintext JSON in `app_data_dir` acceptable for v1.1 ship; Keychain is v1.2 hardening"). PR note required at ship time. |
| T-07-03 | Information Disclosure | Error message body in `map_oauth_error` | mitigate | `map_oauth_error` truncates body to 200 chars via `.chars().take(200).collect()`; the token itself is never substituted into the error string. |
| T-07-04 | Tampering | Corrupt `credentials.json` parse | mitigate | `AuthState::new()` returns `CredentialStore::default()` on `serde_json::from_str` error — covered by `test_corrupt_file_falls_back_to_default` test. |
| T-07-05 | Spoofing | Setup-token format | mitigate | `save_setup_token` rejects any token not starting with `sk-ant-oat01-` before making the API call — covered by unit test in oauth.rs. |
| T-07-SC | Tampering | reqwest 0.12 install | mitigate | `reqwest` is on the RESEARCH.md `## Package Legitimacy Audit` — approved (8+ years old, canonical Rust HTTP client, ~250M monthly downloads via tokio's bundling). |
</threat_model>
