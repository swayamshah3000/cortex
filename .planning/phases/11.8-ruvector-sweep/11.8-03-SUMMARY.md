## Decisions Made
- Verdict is **NO-GO**, not GO-WITH-CAVEAT, because the mismatch is structural (PPR vs. global PageRank), not a minor API-shape gap. Adapting PPR into a global ranking (e.g. via `ppr_multi_seed` with all nodes as uniformly-weighted seeds) is mathematically closer to correct but unvalidated and defeats the crate's sublinear-time selling point — that validation work is out of scope for an audit-only plan and for Plan 08 as currently scoped (implementation, not algorithm R&D).
- No damping factor / iteration count recommendation is given (N/A for NO-GO) — instead, the audit recommends a from-scratch in-house power-iteration PageRank as the v1.3+ path if global entity ranking is still wanted, since Cortex's graph scale (thousands of entities, single-user desktop) doesn't need ruvector-solver's sublinear machinery at all for a straightforward power-iteration implementation.
