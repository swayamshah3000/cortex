<objective>
Wire the hyperbolic HNSW index built in Phase 10 into `search_documents_impl` — the search path that currently ignores it (D-19/D-20/D-21/D-22). Delivers Phase 10 SC5 finally. No new crate dependencies — hyperbolic-hnsw is already in Cargo.toml and state.hyp_index is already populated after each recluster().

Purpose: Phase 10 built + benchmarked the hyperbolic index but never wired the query path. Users who filter by parent space today get flat HNSW + post-filter (correct results, but no hierarchy-aware ranking). This plan makes parent-scoped queries route through hyp_search when available; falls back silently to flat + membership filter when not.

Output:
- New `hyp_search` helper in `search/query.rs` that consumes `state.hyp_index` + `state.hyp_id_to_space` when `SearchFilters.space_id.is_some()`
- New `space_descendant_candidates` helper in `search/filters.rs` that expands a parent space_id into `HashSet<String>` of doc_ids for candidate narrowing (used by both hyp and flat fallback)
- Existing tests continue to pass; new tests cover the two branches
</objective>
