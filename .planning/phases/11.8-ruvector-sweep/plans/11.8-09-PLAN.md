<objective>
When active AI provider is `local-ruvllm`, bump Pass 2 (LLM entity refinement) + Pass 3 (relation extraction) semaphore capacities from 1 → 4 (D-23). Local inference has no rate limit and benefits from parallelism up to CPU-core count; cloud APIs stay at 1 to preserve the UI-responsive behavior established by commits 95a3009 + bbcaac6.

Purpose: The current semaphore cap = 1 exists ONLY because cloud LLM calls compete with the UI for tokio's blocking thread pool and cause preview / get_document lag. With ruvllm running on-device via Metal, there's no such contention — throughput scales linearly with parallelism until CPU saturates. Bumping cap = 4 lets backfill saturate the local model 4-6× faster.

- **Detection MUST be non-fatal.** If `get_active_credential` returns Err or None, default to cap=1 (safe). Never panic in the constructor path.
