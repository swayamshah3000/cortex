<objective>
Compute entity-level PageRank over the TripleStore graph to power a smarter "prominence" signal in Sidebar top-5, ontology panel, and future ranking (D-14/D-15/D-16/D-17/D-18). Ranks entities by graph importance: a Person with `owns` edges to Property + Vehicle + Investment ranks above a Person mentioned once.

Purpose: doc_count is a coarse prominence signal — a person mentioned 500 times in one boring document scores identically to a person with 30 first-class relationships. PageRank over the triple-store forward index captures the actual "who matters" signal the user intuits.

Output:
- If 11.8-03-AUDIT.md verdict == NO-GO: `11.8-08-DEVIATION.md` written; NO code changed; plan exits (fallback: `doc_count DESC` remains prominence signal — existing behavior).

</objective>
