# Phase 11.8: RuVector Integration Sweep — Local LLM + Attention Re-rank + Entity PageRank + Hyperbolic Search - Context

**Gathered:** 2026-07-09
**Status:** Ready for planning
**Trigger:** User directive: "leverage all the possible features of ruvector." Audit narrowed to top-4 highest-ROI integrations.

<domain>
## Phase Boundary

Phase 11.8 integrates 4 ruvector crates that deliver real user-visible value. **Each integration is gated on a fresh crate audit** — Phase 9/10/12/13 taught us ruvector crate names sometimes don't match cortex-relevant functionality. Planner audits `crates/{name}/{README.md, src/lib.rs}` BEFORE writing implementation plans; if a crate is unfit, that integration is skipped w/ documented deviation.

### What Phase 11.8 delivers (subject to per-crate audit gate)

1. **ruvllm — local LLM inference (biggest win).** Replace 90% of Anthropic API calls with local Metal/CUDA inference. Backfill Pass 2 + Pass 3 + RAG chat run on-device. Zero API cost, zero rate limits, zero privacy leak, works offline. Integration: alt provider `local-ruvllm` in Phase 7 credential store; ai_request() routes to ruvllm handler when active. First-run downloads model weights (~4-7GB Qwen or Llama).

2. **ruvector-attention — post-cosine search re-ranking.** After HNSW top-K, run one of the 46 attention mechanisms (Flash Attention / Optimal Transport / etc.) over the top-20 candidates to rerank by learned relevance. Better semantic quality than pure cosine.

3. **ruvector-solver — entity PageRank over TripleStore.** Rank entities by "graph importance" — a Person node with many `owns` edges to Property + Vehicle + Investment ranks above a Person mentioned once. Replaces doc_count as prominence signal in Sidebar + Ontology panel. Sublinear PageRank runs on triple-store graph in O(log n).

4. **ruvector-hyperbolic-hnsw wiring — existing gap.** Phase 10 built + benchmarked the hyperbolic index but no search path uses it. Wire into search_documents_impl: when SearchFilters.parent_space_id is Some, use hyperbolic path; else flat. Delivers Phase 10 SC5 finally.

### Out of scope (defer to v1.3)

- ruvector-temporal-tensor vector compression (nice-to-have; not blocking at current 2K-doc scale)
- rupixel visual doc search (~800 LoC + PDF renderer, own phase)
- ruvector-metrics observability (not user-facing)
- ruvector-verified proof-carrying ops (correctness insurance, marginal at MVP)
- ruvector-sparse-inference (subsumed by ruvllm if ruvllm audit passes)
- Any distributed-DB crates (single-user desktop)

</domain>

<decisions>
## Implementation Decisions

### Integration 1: ruvllm Local LLM

- **D-01: Planner audits FIRST.** Reads `crates/ruvllm/{README.md, src/lib.rs}` + top-level examples. Confirms: (a) Rust crate loadable via Cargo path dep; (b) Anthropic-style chat completion API surface exists or can be adapted; (c) Metal backend available on Apple Silicon (user is on macOS ARM); (d) streaming supported (for RAG chat).
- **D-02: If unfit** — planner writes 11.8-01-DEVIATION.md documenting exactly what's missing, skips this integration, continues with others.
- **D-03: If fit — add as new provider in Phase 7 abstraction.** New provider slug `local-ruvllm`. New `ai/ruvllm.rs` module. Enrolls in `normalize_provider_name` + `ai_request` routing. No OAuth (local).
- **D-04: Model download UX.** First "Connect" click opens dialog: model selection (Qwen 2.5 7B / Llama 3.2 3B), disk space warning (~4-7GB), download-progress bar. Persist to `app_data_dir/ruvllm-models/`.
- **D-05: Streaming for RAG chat.** ruvllm streaming variant wired to Phase 11.7's `ai_request_stream()`.
- **D-06: Fallback.** If ruvllm init fails at runtime (Metal unavailable / model missing), toast + fall back to previous active provider. Never leave user with broken inference.
- **D-07: Settings surface.** Provider card shows: model name + size, disk usage, inference speed benchmark ("last query 1.2s"), "Redownload model" button.

### Integration 2: ruvector-attention Search Re-rank

- **D-08: Planner audits FIRST.** Reads `crates/ruvector-attention/{README.md, src/lib.rs}`. Confirms: (a) crate exposes a re-rank API taking query embedding + candidate embeddings + attention scores; (b) at least one mechanism is CPU-friendly (no GPU requirement); (c) attention scores are actionable (interpretable as relevance).
- **D-09: If unfit** — deviation, skip.
- **D-10: If fit — add re-rank stage AFTER cosine top-K.** New `search/attention_reranker.rs` module. Consumed by `search_documents_impl` after HNSW returns top-20; reranker returns reordered top-N.
- **D-11: Attention mechanism choice.** Planner picks from the 46 based on: CPU-friendly, semantic-relevance-oriented (not e.g. compression), lightweight (< 50ms per query on 20 candidates).
- **D-12: Recency + Jaccard preserved.** Re-rank uses cosine + attention as base; final formula becomes `0.5 * cosine + 0.3 * attention + 0.15 * jaccard + 0.05 * recency` (weights tunable in Settings v1.3).
- **D-13: Feature-flag opt-in.** Settings toggle "Enhanced search re-ranking (experimental)". Default OFF until user validates quality on their corpus.

### Integration 3: ruvector-solver Entity PageRank

- **D-14: Planner audits FIRST.** Reads `crates/ruvector-solver/{README.md, src/lib.rs}`. Confirms: (a) exposes PageRank on user-supplied sparse graph; (b) accepts `Vec<(subject_id, object_id, weight)>` edge list; (c) returns Vec<(node_id, rank_score)>.
- **D-15: If unfit** — deviation, skip; fallback to `doc_count DESC` for prominence.
- **D-16: If fit — new `graph/entity_pagerank.rs` module.** Runs on TripleStore's forward-index. Edge weight = triple provenance doc count. Result: HashMap<entity_id, pagerank_score>.
- **D-17: Cache + refresh.** PageRank scores computed after each TripleStore batch write (throttled to every 100 triples OR 10 min). Stored in `app_data_dir/entity_pagerank.json` alongside TripleStore.
- **D-18: Prominence signal replaces doc_count.** `get_prominent_entities` IPC (from Phase 11.6) uses PageRank when available. Sidebar top-5 sorted by PageRank descending.

### Integration 4: ruvector-hyperbolic-hnsw Wiring

- **D-19: No audit needed** — Phase 10 already validated crate + benchmarked. Just wire the search path.
- **D-20: Detect parent-space filter.** In `search_documents_impl`, if `SearchFilters.parent_space_id.is_some()`, route to `hyp_search()` helper using `state.hyp_index` + `state.hyp_id_to_space`.
- **D-21: Fallback preserved.** If hyp_index is empty (bootstrap edge case) → fall back to flat HNSW filtered by parent Space membership.
- **D-22: Perf assertion.** Integration test: parent-scoped search on 10k-doc synthetic corpus completes ≤ 2× flat search time. Documented in SUMMARY.

### Cross-Cutting: Provider-Aware Concurrency

- **D-23:** Current Pass 2 + Pass 3 semaphores at 1 (UI-responsiveness). When active provider = `local-ruvllm`, bump both to 4 — local inference has no rate limit and benefits from parallelism up to CPU core count. When active provider = cloud, stay at 1.

<specifics>
## Specific Ideas

- **ruvllm is the killer feature.** If it works: cortex becomes fully local + private + free + offline. Every user delight point goes up.
- **Attention re-rank ships as opt-in experimental.** Users try it, we learn, promote to default in v1.3 if signal is positive.
- **PageRank matches user's intuition of "who matters"** — Alex Doe with 500 docs + owns relationships to Property, Vehicle, Investment ranks above someone in one doc.
- **Hyperbolic wiring is the smallest change but shipped the longest ago** (Phase 10). Ship for closure.
- **Audit gate is the safety net** — Phase 9's ruvector-cluster (gossip protocol, not clustering) taught us this discipline.

</specifics>
