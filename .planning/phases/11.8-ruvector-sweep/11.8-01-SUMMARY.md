## Decisions Made
- **GO verdict.** `LlmBackend::generate()` and `generate_stream_v2()`/`generate_stream_async()` are directly callable; `apply_chat_template(&[ChatMessage])` handles system+multi-turn message flattening (matches cortex's `system_prompt` + `Vec<ServiceMessage>` shape).
- **Feature flags:** `inference-metal, gguf-mmap, parallel, accelerate, async-runtime` — minimal set for Metal-accelerated inference + streaming on Apple Silicon, deferring ANE/hybrid-pipeline complexity to v1.3.
- **Default model:** Qwen2.5-7B-Instruct Q4_K_M (best throughput/quality per README's own M4 Pro benchmarks: 95 tok/s decode, 4.2GB memory). Llama 3.2 3B Instruct offered as the lightweight fallback in the D-04 model-picker dialog.
- **load_model() handles HF Hub download implicitly** — passing an HF repo id string triggers `load_from_hub()` automatically, so Plan 06 does not need a separate downloader integration for the basic case (though `hub::ModelDownloader` + `DownloadProgress`/`ProgressCallback` are available if an explicit progress-bar UX is wanted).
