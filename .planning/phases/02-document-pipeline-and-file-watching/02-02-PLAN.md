  <action>
  Create `src-tauri/src/pipeline/embedder.rs`:

  **EmbeddingService struct:**
  ```rust
  pub struct EmbeddingService {
      model: std::sync::Mutex<fastembed::TextEmbedding>,
      pub dimensions: usize,
  }
  ```

  Use `std::sync::Mutex` (not tokio::sync::Mutex) because fastembed's `embed()` is synchronous and will be called inside `spawn_blocking`. The `std::sync::Mutex` avoids requiring async lock in sync context.

  **new_local() constructor:**
  - Call `fastembed::TextEmbedding::try_new(fastembed::InitOptions::new(fastembed::EmbeddingModel::AllMiniLML6V2))`
  - Map errors to `AppError::Embedding(e.to_string())`
  - Set `dimensions: 384`
  - NOTE: First call downloads ~90MB model to `~/.cache/fastembed/`. This is expected behavior.

  **embed_text(&self, text: &str) -> Result<Vec<f32>, AppError>:**
  - Truncate input text to 2000 chars using `truncate_to_chars()` helper
  - Lock the mutex: `self.model.lock().map_err(|e| AppError::Embedding(e.to_string()))?`
  - Call `model.embed(vec![chunk.as_str()], None)` -- fastembed takes `Vec<&str>` and batch size
  - Map errors to `AppError::Embedding`
  - Return the first (and only) embedding from the result vec
  - If result is empty, return `AppError::Embedding("Empty embedding result")`

  **truncate_to_chars(text: &str, max_chars: usize) -> String:**
  - If `text.len() <= max_chars`, return `text.to_string()`
  - Otherwise `text.chars().take(max_chars).collect()`

  **DPIP-07 (API embeddings):** Add a comment/doc note that API embeddings (OpenAI text-embedding-3-small, 1536-dim) will use ruvector-core's `ApiEmbedding::openai()` which is already wired from Phase 1. The `documents_1536` collection already exists. The EmbeddingService focuses on LOCAL embeddings. API embedding path will be activated via settings toggle in Phase 4.

  - Test that `truncate_to_chars` truncates correctly

  <files>

  </files>
  <action>
  Create `src-tauri/src/pipeline/entities.rs`:

  **EntityExtractor struct:**
  ```rust
  pub struct EntityExtractor {
      date_re: Regex,
      amount_re: Regex,
      email_re: Regex,
      person_re: Regex,
  }
  ```

  **new() constructor:**
  Compile all regex patterns once:
  - `date_re`: Match ISO dates (`\d{4}-\d{2}-\d{2}`), US dates (`\d{1,2}/\d{1,2}/\d{2,4}`), and written dates (`Jan 15, 2024` etc): `r"\b(\d{4}-\d{2}-\d{2}|\d{1,2}/\d{1,2}/\d{2,4}|(?:Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)[a-z]*\.?\s+\d{1,2},?\s+\d{4})\b"`
  - `amount_re`: Match dollar/euro/pound amounts: `r"\b(?:USD\s*)?[$\u{00a3}\u{20ac}]\s*[\d,]+(?:\.\d{2})?\b|\b[\d,]+(?:\.\d{2})?\s*(?:USD|EUR|GBP)\b"`
  - `email_re`: Standard email pattern: `r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b"`
  - `person_re`: Capitalized two-word sequences (heuristic for names): `r"\b([A-Z][a-z]+\s+[A-Z][a-z]+)\b"`

  **extract(&self, text: &str) -> Vec<ExtractedEntity>:**
  - Run each regex against the text
  - For each date match: `ExtractedEntity { label: "Date", value: match_str, entity_type: "date" }`
  - For each amount match: `ExtractedEntity { label: "Amount", value: match_str, entity_type: "amount" }`
  - For each email match: `ExtractedEntity { label: "Email", value: match_str, entity_type: "person" }` (emails are person-adjacent)
  - For each person_re match: `ExtractedEntity { label: "Person/Org", value: match_str, entity_type: "person" }`
  - Deduplicate by value: `entities.sort_by(|a, b| a.value.cmp(&b.value)); entities.dedup_by(|a, b| a.value == b.value);`
  - Truncate to 20 entities max: `entities.truncate(20);`
  - Return the entities vec

  Use `crate::types::ExtractedEntity` for the return type.

  Add `pub mod entities;` to pipeline/mod.rs.

  - Test written date: "January 15, 2024" -> entity with value matching, type "date"
  - Test amount extraction: "Total: $1,234.56" -> entity with value "$1,234.56", type "amount"
  - Test person extraction: "Contact John Smith" -> entity with value "John Smith", type "person"
  - Test deduplication: same value appearing twice produces only one entity
  - Test truncation: text with 30+ entities produces max 20
  - Test empty text: returns empty vec
