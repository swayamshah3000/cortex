<objective>
Build the file watching infrastructure: WatcherRegistry for persistent folder configuration and the watcher worker background task that monitors folders for changes via notify-debouncer-mini.

Purpose: The watcher layer detects file changes in real time and routes them to the indexing pipeline. The registry persists user-configured folders, exclusions, and file type toggles across app restarts. The worker task runs for the lifetime of the app.

Output: `watcher/registry.rs` with `WatcherRegistry`, `watcher/worker.rs` with `spawn_watcher_task` and command handling.
</objective>

  **watcher/registry.rs:**

  Define `WatchedFolderConfig`:
  ```rust
  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct WatchedFolderConfig {
      pub id: String,
      pub path: String,
      pub enabled_types: Vec<String>,       // ["pdf", "docx", "txt", "md", "xlsx", "csv"]
      pub excluded_patterns: Vec<String>,   // ["node_modules", ".git", "target"]
      pub is_paused: bool,
      pub document_count: u32,
      pub last_scan: Option<String>,        // ISO 8601
  }
  ```

  Define `WatcherRegistry`:
  ```rust
  #[derive(Debug, Default, Serialize, Deserialize)]
  pub struct WatcherRegistry {
      pub folders: HashMap<String, WatchedFolderConfig>,
  }
  ```

  Implement methods:

  **load(registry_path: &PathBuf) -> Self:**
  - Try `std::fs::read_to_string(registry_path)`, then `serde_json::from_str()`
  - On any error, return `Self::default()` (empty registry -- first run)

  **save(&self, registry_path: &PathBuf) -> Result<(), AppError>:**
  - `serde_json::to_string_pretty(self)?` then `std::fs::write(registry_path, json)?`
  - Map serde errors to `AppError::Internal`

  **add_folder(&mut self, path: String) -> WatchedFolderConfig:**
  - Generate ID: `uuid::Uuid::new_v4().to_string()`
  - Create config with default enabled_types: `["pdf", "docx", "txt", "md", "xlsx", "csv", "xls", "ods"]`
  - Default excluded_patterns: `["node_modules", ".git", "target", "__pycache__", ".DS_Store"]`
  - `is_paused: false`, `document_count: 0`, `last_scan: None`
  - Insert into `self.folders` and return clone

  **remove_folder(&mut self, folder_id: &str) -> bool:**
  - Remove from HashMap, return whether it existed

  **is_excluded(&self, folder_id: &str, path: &Path) -> bool:**
  - Get config for folder_id (return false if not found)
  - Check if filename starts with '.' (hidden files)
  - Check if any path component matches an excluded_pattern (substring match in path string)
  - Return true if any exclusion matches

  **is_type_enabled(&self, folder_id: &str, ext: &str) -> bool:**
  - Get config for folder_id (return false if not found)
  - If `enabled_types` is empty, return true (all types enabled)
  - Otherwise check if ext (lowercased) is in `enabled_types`

  **find_folder_for_path(&self, file_path: &Path) -> Option<&WatchedFolderConfig>:**
  - Iterate all folders, find one whose `path` is a prefix of `file_path`
  - Return the matching config (needed by the worker to determine which folder a file event belongs to)

  Add `mod watcher;` to lib.rs (after `mod pipeline;`).

  **Tests:**
  - Test load from nonexistent file returns empty registry
  - Test save then load round-trips correctly
  - Test add_folder creates config with defaults
  - Test remove_folder returns true for existing, false for nonexistent
  - Test is_excluded: ".git/config" excluded, "src/main.rs" not excluded, ".hidden_file" excluded
  - Test is_type_enabled: "pdf" enabled by default, "exe" not in defaults
  - Test find_folder_for_path matches correct folder

  ```rust
  pub enum WatcherCommand {
      AddFolder { path: String, folder_id: String },
      RemoveFolder { folder_id: String, path: String },
      Pause,
      Resume,
      Shutdown,
  }
  ```

  **Define IndexProgress event payload** (for Tauri event emission):
  ```rust
  #[derive(Clone, serde::Serialize)]
  pub struct IndexProgress {
      pub file_path: String,
      pub status: String,   // "indexing", "indexed", "skipped", "error"
      pub doc_id: Option<String>,
      pub error: Option<String>,
      pub folder_id: Option<String>,
  }
  ```

  **Define FileEvent** (internal type for the channel between notify and the processing loop):
  ```rust
  struct FileEvent {
      path: PathBuf,
      kind: FileEventKind,
  }
  enum FileEventKind {
      CreateOrModify,
      Remove,
  }
  ```

  **spawn_watcher_task function:**
  ```rust
  pub fn spawn_watcher_task(
      app_handle: tauri::AppHandle,
      engine: Arc<tokio::sync::Mutex<CortexEngine>>,
      embedding_service: Arc<EmbeddingService>,
      registry: Arc<std::sync::Mutex<WatcherRegistry>>,
      registry_path: PathBuf,
      mut cmd_rx: tokio::sync::mpsc::Receiver<WatcherCommand>,
  )
  ```

  Implementation:
  1. Spawn via `tauri::async_runtime::spawn(async move { ... })`
  2. Inside the task:
     - Create a `tokio::sync::mpsc::channel::<FileEvent>(256)` for file events
     - Create debouncer: `notify_debouncer_mini::new_debouncer(Duration::from_millis(300), move |res| { ... })`. In the callback, convert debouncer events to `FileEvent` messages and send via the channel using `blocking_send`.
     - Load registry, iterate all non-paused folders, call `debouncer.watcher().watch(Path::new(&config.path), RecursiveMode::Recursive)` for each
     - Enter main `loop` with `tokio::select!`:
       - **File events branch**: Receive from file event channel. For each event:
         - Check if excluded (via registry)
         - Check if file type is enabled (via registry)
         - If CreateOrModify: Placeholder for indexing (emits "indexing" then "indexed" events). Real indexing wired in Plan 05.
         - If Remove: Placeholder for de-indexing (emit "removed" event)
       - **Command branch**: Receive from `cmd_rx`:
         - `AddFolder { path, folder_id }`: Call `debouncer.watcher().watch()` on the new path
         - `RemoveFolder { folder_id, path }`: Call `debouncer.watcher().unwatch()` on the path
         - `Pause`: Unwatch all folders
         - `Resume`: Re-watch all non-paused folders
         - `Shutdown`: break the loop

  IMPORTANT: Use `use tauri::Emitter;` to bring emit() into scope. Call `app_handle.emit("index-progress", payload)` for event emission.

  IMPORTANT: The debouncer variable MUST be kept alive for the lifetime of the loop. If it's dropped, watching stops. Store it as a local variable in the task closure, not in a nested scope.

  IMPORTANT: notify-debouncer-mini callback receives `DebounceEventResult` which is `Result<Vec<DebouncedEvent>, Vec<notify::Error>>`. Each `DebouncedEvent` has `path: PathBuf` and `kind: DebouncedEventKind` (either `Any` or `AnyContinuous`). Both kinds should be treated as CreateOrModify.

  For FWAT-02 (polling fallback): Add a comment documenting that notify-rs's `PollWatcher` can be substituted if `RecommendedWatcher` fails on certain filesystems (e.g., NFS, FUSE). This is a future enhancement -- for now, `RecommendedWatcher` (the default in notify-debouncer-mini) covers macOS FSEvents, Linux inotify, and Windows ReadDirectoryChanges.

  - Test that IndexProgress serializes to JSON correctly
  - Note: Full integration testing of the watcher task requires a running Tauri app -- functional testing deferred to Plan 05's integration verification
