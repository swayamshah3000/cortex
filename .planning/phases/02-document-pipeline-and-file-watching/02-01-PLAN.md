<objective>
Build the document parsing dispatcher and content hasher -- the two CPU-bound, library-dependent modules that transform raw files into text and detect changes.

Purpose: Every file type (PDF, DOCX, TXT, MD, XLSX, CSV) needs a text extraction path, and every file needs a content hash for change detection. These are foundational building blocks used by the indexer (Plan 03) and watcher (Plan 05).

Output: `pipeline/parser.rs` with `parse_document()`, `pipeline/hasher.rs` with `content_hash()`, updated Cargo.toml with all Phase 2 dependencies, extended AppError with new variants.
</objective>

  <action>
  Create `src-tauri/src/pipeline/` directory with three files:

  **pipeline/mod.rs** -- re-exports:
  ```rust
  pub mod parser;
  pub mod hasher;
  ```

  **pipeline/parser.rs** -- Type-dispatched text extraction:
  - Define `ParsedDocument { text: String, title: String, doc_type: String }`
  - Implement `pub fn parse_document(path: &Path) -> Result<ParsedDocument, AppError>` that:
    - Reads the file extension (lowercase)
    - Dispatches to the right parser:
      - `"pdf"` -> `pdf_extract::extract_text_from_mem(&bytes)` (read file to bytes first)
      - `"docx"` | `"doc"` -> `docx_rust::DocxFile::from_file(path)?.parse()?` then iterate paragraphs extracting text content. Use `extract_docx_text()` helper that walks the document body paragraphs and concatenates text runs.
      - `"txt"` | `"md"` -> `std::fs::read_to_string(path)`
      - `"csv"` -> `std::fs::read_to_string(path)` (CSV is already text)
      - `"xlsx"` | `"xls"` | `"ods"` -> calamine: `open_workbook_auto(path)`, iterate all sheets, all rows, collect String/Float/Int cells, join with spaces/newlines
      - `"png"` | `"jpg"` | `"jpeg"` | `"tiff"` -> Return `Err(AppError::Parse("OCR not available — image indexing requires tesseract (coming soon)".to_string()))`. This is the DPIP-05 placeholder.
      - Other extensions -> `Err(AppError::Parse(format!("Unsupported file type: {}", ext)))`
    - Sets `title` from `path.file_name()`
    - Sets `doc_type` from the extension string
  - All library errors mapped via `.map_err(|e| AppError::Parse(e.to_string()))`

  IMPORTANT for docx-rust: The `docx_rust::Docx` struct has a `body` field containing `BodyContent` items. Walk `doc.document.body.content` items, match on `BodyContent::Paragraph(p)`, then iterate `p.content` for `ParagraphContent::Run(r)`, then `r.content` for `RunContent::Text(t)` to get `t.text`. If the API differs from this, adapt accordingly -- the goal is extracting all text content from paragraphs.

  **pipeline/hasher.rs** -- SHA-256 content hash:
  - Implement `pub fn content_hash(path: &Path) -> Result<String, AppError>`:
    - Open file with `std::fs::File::open(path)?`
    - Create `sha2::Sha256::new()` hasher
    - Read in 4KB chunks via `file.read(&mut buffer)`, feed to `hasher.update()`
    - Return `format!("{:x}", hasher.finalize())`
  - Import `sha2::{Sha256, Digest}` and `std::io::Read`

  **lib.rs** -- Add `mod pipeline;` to the module declarations (after `mod types;`).

  Add unit tests to both parser.rs and hasher.rs:
  - parser.rs: test that `parse_document` on a ".txt" temp file returns the file content, test that unsupported extension returns error
  - hasher.rs: test that `content_hash` on a temp file returns a 64-char hex string, test that identical content produces identical hash, test that different content produces different hash
