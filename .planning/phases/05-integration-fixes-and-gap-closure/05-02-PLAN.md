Add a useEffect in AppShell.tsx that listens for Tauri "index-progress" events and bridges them to the useIndexingStore Zustand store.

1. Import `isTauri` from `@/lib/utils` (if not already imported).
2. Import `useIndexingStore` from the appropriate store path.
3. Add a useEffect (with empty dependency array) that:
   a. Returns early if `!isTauri()`
   b. Dynamically imports `@tauri-apps/api/event` to get `listen`
   c. Calls `listen<IndexProgress>("index-progress", (event) => { ... })`
   d. In the handler: call `useIndexingStore.getState().setProgress(event.payload)` to update the store.
      If the status is "complete" or "error" and it indicates the overall scan is done, optionally call `clearProgress()` after a short delay.
   e. Stores the unlisten promise and calls it in the cleanup function.

The pattern should match this structure:
```tsx
useEffect(() => {
  if (!isTauri()) return;
  let unlisten: (() => void) | undefined;
  (async () => {
    const { listen } = await import("@tauri-apps/api/event");
    unlisten = await listen("index-progress", (event: any) => {
      useIndexingStore.getState().setProgress(event.payload);
    });
  })();
  return () => { unlisten?.(); };
}, []);
```

Why: This is the missing bridge between Tauri backend events and the Zustand store that drives the TopBar indexing indicator.
