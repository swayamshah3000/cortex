In src-tauri/src/lib.rs:
1. After the line `let indexer = Arc::new(DocumentIndexer::new());` (around line 52), add a call to rebuild the path index from persisted vectors:
```rust
{
    let engine_guard = engine_arc.blocking_lock();
    if let Err(e) = indexer.rebuild_path_index(&engine_guard) {
        eprintln!("Warning: failed to rebuild path index: {}", e);
    }
}
```
This is inside the setup closure which runs synchronously before the app starts, so blocking_lock is safe here.

In src-tauri/src/commands/settings.rs:

If AppState does not have app_data_dir, use a well-known path like `dirs::data_dir().unwrap().join("com.cortex.app").join("settings.json")` or add an `app_data_dir` field. Check what AppState contains first by reading `src-tauri/src/state.rs`.
4. Rewrite `get_settings` to: try reading `settings.json` from app_data_dir, deserialize with serde_json. If file doesn't exist or parse fails, return the current hardcoded defaults.

Why: rebuild_path_index populates the in-memory path->id map from persisted vectors, preventing duplicate embeddings. Settings persistence via JSON file is the simplest reliable approach for a desktop app.
