## Spacing Scale

Cortex uses a 4px base grid. Inherited from Phase 9 contract. **All phase 10 spacing values are strict multiples of 4. No sub-grid values are declared.**

| Token | Value | Usage in Phase 10 |
|-------|-------|-----------------|
| xs | 4px | Chevron-to-name gap in sidebar sub-space row (`gap-1`), sub-count badge vertical padding (`py-1`), breadcrumb separator gap |
| sm | 8px | Sidebar sub-space row horizontal padding (`px-2`), parent context banner icon-to-text gap (`gap-2`), sub-space list indent (`pl-8` = 32px — see below) |
| md | 12px | Sub-space list row padding (`px-3`), "Misc" dashed badge horizontal padding (`px-3`) |
| lg | 16px | SpaceDetailPage parent context banner padding (`px-4 py-2`), sub-space grid column gap (`gap-4`) |
| xl | 24px | SpaceCard inherited card padding (`p-6`), section separator (`space-y-6`) |
| 2xl | 32px | Sub-space list indent from sidebar parent row (rendered as `pl-8`, exact 32px) |

**Exceptions:**
- Touch targets for chevron-only buttons in sidebar: minimum 44×44px wrapper — 44 = 4 × 11, grid-aligned.
- Sidebar sub-space row `py-2` (8px) not `py-2.5` — Phase 10 must not introduce new sub-grid values. Note: existing Sidebar NavLink uses `py-2.5` (Phase 4 inheritance). Sub-space rows use `py-2` to visually distinguish their secondary hierarchy weight.

**No sub-grid values.** Phase 10 does NOT use `py-0.5`, `py-1.5`, `px-2.5`, or `gap-1.5`.

## Typography

Inherited from Phase 9 contract. Phase 10 introduces no new type roles.

| Role | Size | Weight | Line Height | Font | Class | Phase 10 usage |
|------|------|--------|-------------|------|-------|--------------|
| Page title | 24px | 600 | 1.2 | Inter | `.page-title` (`text-2xl font-semibold`) | SpaceDetailPage: space name heading (inherited from Phase 9) |

**Weights declared for phase 10:** exactly **2** — 400 (regular) and 600 (semibold). Matches Phase 9 budget. No 500 (medium) introduced.

**Sub-count format:** The sub-count `(3)` rendered beside the Space name in the sidebar uses `text-xs text-text-tertiary` inline, not a badge element. Example: `Property (3)` where `Property` is `text-sm font-medium` and ` (3)` is `text-xs text-text-tertiary ml-1`.

**Semantic role protection (inherited from Phase 9):** 12px is reserved strictly for non-prose display text (chips, timestamps, sidebar labels, breadcrumb). 14px is reserved for prose. No future role may occupy the 13px slot.

## Visual Hierarchy

**Primary focal point of the Sidebar spaces section (Phase 10 expanded state):** The top-level Space name with sub-count `"Property (3)"`. The sub-space list below is indented and visually subordinate — smaller text, less padding, no color accent.

**Primary focal point of SpaceDetailPage (sub-space view):** The breadcrumb `Spaces / Property / Tax` establishes context immediately. The parent context banner below it ("Sub-space of Property") reinforces hierarchy without competing with the heading.

**"Misc" sub-space card** must not be confused with a labeled sub-space. The dashed border communicates "system-generated grouping" vs "AI-labeled cluster." It renders last in the sub-space grid (after all labeled sub-spaces).

**SpaceLabelingIndicator reuse:** When sub-space labeling is in flight (triggered after sub-clustering completes), the existing `SpaceLabelingIndicator` chip from Phase 9 appears in the SpacesPage header — no new loading UI needed. The chip's copy changes to "Labeling sub-spaces {N}/{M}" during a sub-space-only labeling run.

## Control Specifications

### 1. Sidebar — Space Row with Chevron + Inline Sub-Space Expand (HSPC-04, D-13, D-14, D-17)

**When to show chevron:** Only for top-level spaces that have `subSpaceIds.length > 0`. Spaces with no sub-spaces show no chevron.

**When to show sub-count:** Always for top-level spaces with sub-spaces, inline after the space name.

**Row structure (expanded sidebar, not collapsed):**

```tsx
<Collapsible
  open={expandedSpaceIds.has(space.id)}
  onOpenChange={() => toggleSpaceExpanded(space.id)}
>
  <div className="flex items-center gap-1 rounded-md hover:bg-bg-tertiary group">
    {/* Color dot */}
    <div className="h-2 w-2 rounded-full flex-shrink-0 ml-3" style={{ backgroundColor: space.color }} />

    {/* Space name + sub-count as Link (navigates to /spaces/:id) */}
    <Link
      to={`/spaces/${space.id}`}
      className="flex-1 flex items-center py-2 text-sm font-medium text-text-secondary hover:text-text-primary truncate"
      onClick={(e) => e.stopPropagation()} // prevent collapsible toggle
    >
      <span className="truncate">{space.name}</span>
      {space.subSpaceIds.length > 0 && (
        <span className="ml-1 text-xs text-text-tertiary flex-shrink-0">
          ({space.subSpaceIds.length})
        </span>
      )}
    </Link>

    {/* Chevron toggle button — only when sub-spaces exist */}
    {space.subSpaceIds.length > 0 && (
      <CollapsibleTrigger asChild>
        <button
          className="p-1 rounded hover:bg-bg-tertiary text-text-tertiary hover:text-text-secondary transition-colors opacity-0 group-hover:opacity-100"
          aria-label={expandedSpaceIds.has(space.id) ? "Collapse sub-spaces" : "Expand sub-spaces"}
          style={{ minWidth: 44, minHeight: 44 }}
        >
          <ChevronRight
            size={14}
            className={cn("transition-transform", expandedSpaceIds.has(space.id) && "rotate-90")}
          />
        </button>
      </CollapsibleTrigger>
    )}
  </div>

  {/* Sub-space list */}
  <CollapsibleContent>
    <div className="pl-8 space-y-0.5 pb-1">
      {subSpaces.map((sub) => (
        <Link
          key={sub.id}
          to={`/spaces/${sub.id}`}
          className="flex items-center gap-2 px-3 py-2 rounded-md text-xs text-text-tertiary hover:bg-bg-tertiary hover:text-text-secondary transition-colors"
        >
          <div className="h-1.5 w-1.5 rounded-full flex-shrink-0" style={{ backgroundColor: sub.color }} />
          <span className="truncate">{sub.name}</span>
        </Link>
      ))}
    </div>
  </CollapsibleContent>
</Collapsible>
```

**Spec:**
- Top 5 spaces by `documentCount` (change from current top 6) — per HSPC-04
- Chevron: `ChevronRight` 14px, rotates 90° when open (`rotate-90` CSS class). CSS `transition-transform` duration 150ms.
- Chevron button: 44×44px minimum touch target, ghost style, `opacity-0 group-hover:opacity-100`. Chevron is ONLY visible on row hover (visibility affordance — reveals capability without visual noise when not needed).
- Sub-count: `"({n})"` inline after name, `text-xs text-text-tertiary`, `ml-1`. Not a badge. Not shown for spaces with 0 sub-spaces.
- Sub-space list indent: `pl-8` (32px). Each sub-space row: 8px dot + 8px gap + name. Row height: `py-2` (8px top + 8px bottom).
- Sub-space dot: 6px (`h-1.5 w-1.5`) vs parent dot 8px (`h-2 w-2`) — smaller dot reinforces hierarchy.
- Expanded state persisted in `useSidebarStore.expandedSpaceIds: Set<string>`. Toggle on chevron click. NOT persisted to localStorage (resets on restart).
- Clicking the space name navigates to `/spaces/:id` (page navigation). Clicking the chevron toggles expand/collapse in place (no navigation). These are two distinct interaction targets.
- When sidebar is collapsed (`isCollapsed === true`): hide chevron, sub-count, and sub-space list entirely. Collapsed sidebar shows only color dot.

### 4. SpaceDetailPage — Sub-Space Grid (HSPC-01, HSPC-02, D-16)

**Current implementation:** `space.subSpaces` is already rendered in `SpaceDetailPage` as an inline `SubSpaceCard` grid (lines 302-312 in existing file). Phase 10 upgrades this by:

1. Using extracted `SubSpaceCard` component (from new file)
2. Sorting sub-spaces: labeled sub-spaces first (sorted by `documentCount` desc), "Misc" sub-space last
3. Adding shimmer state when sub-space labeling is in flight

**"Misc" identification:** A sub-space where `space.name === "Misc"` (exact string, case-sensitive, set by backend `subspace_detector.rs`) renders with `border-dashed` on the left border.

```tsx
{/* Sub-spaces section */}
{space.subSpaces.length > 0 && (
  <div className="space-y-3">
    <h2 className="section-header text-text-primary">Sub-Spaces</h2>
    <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
      {sortedSubSpaces.map((sub) => (
        <SubSpaceCard key={sub.id} space={sub} isMisc={sub.name === "Misc"} />
      ))}
    </div>
  </div>
)}
```

**Sort logic:**

```typescript
const sortedSubSpaces = useMemo(() => {
  const labeled = space.subSpaces.filter(s => s.name !== "Misc").sort((a, b) => b.documentCount - a.documentCount);
  const misc = space.subSpaces.filter(s => s.name === "Misc");
  return [...labeled, ...misc];
}, [space.subSpaces]);
```

### 7. Sub-Space Labeling Loading State (LLML-05 reuse, D-16)

When sub-space labeling is in flight (new sub-cluster has been detected and LLM is labeling sub-spaces), the **existing `SpaceLabelingIndicator`** from Phase 9 is reused with copy variant.

**Copy for sub-space labeling run:**

| Status | Copy |
|--------|------|
| `labeling` | `"Labeling sub-spaces {processed}/{total}"` |
| `complete` | `"Sub-space labels ready"` |
| `error` | `"Sub-space labeling failed"` |

This is a copy-only change on the indicator. No component modification needed — the indicator reads copy from the backend `SpaceLabelingProgress` event payload.

**On SpaceDetailPage during sub-space labeling:** If a sub-space is still being labeled (its `labelStatus === 'generating'`), render `<Skeleton className="h-5 w-28" />` in its position in the sub-space grid, replacing the SubSpaceCard link with a non-interactive skeleton card:

```tsx
{sub.labelStatus === 'generating' ? (
  <div className="card p-4 border-l-4" style={{ borderLeftColor: sub.color }}>
    <div className="flex items-center gap-3">
      <div className="p-2 rounded-lg bg-accent-subtle">
        <Skeleton className="h-4 w-4" />
      </div>
      <div className="space-y-1.5">
        <Skeleton className="h-4 w-28" />
        <Skeleton className="h-3 w-16" />
      </div>
    </div>
  </div>
) : (
  <SubSpaceCard space={sub} isMisc={sub.name === "Misc"} />
)}
```

## Data Type Extensions

Phase 10 adds two fields to the `Space` type in `client/lib/types.ts`:

```typescript
interface Space {
  // ... existing fields (Phase 9) ...

  // -------------------------------------------------------------------------
  // Phase 10 Plan 01: Hierarchical Spaces fields
  // Mirrors Rust Space struct additions (all #[serde(default)] on Rust side).
  // Frontend must treat absent / undefined values as follows:
  //   depth       → undefined | 0 = top-level space
  //   subSpaceIds → undefined | [] = no sub-spaces
  // -------------------------------------------------------------------------
  /** 0 for top-level spaces, 1 for sub-spaces. Max depth = 2 (gated by backend). */
  depth?: number;
  /** IDs of direct child sub-spaces. Empty array for sub-spaces and spaces < 50 docs. */
  subSpaceIds?: string[];
}
```

**Note:** `parentId?: string` already exists on the `Space` interface (Phase 4). `subSpaces: Space[]` (nested array) also already exists. Phase 10 adds `depth` and `subSpaceIds` for flat-list operations. The `subSpaces` nested array remains the source for sub-space rendering on SpaceDetailPage; `subSpaceIds` is used for count display on SpaceCard and sidebar.

**`useSidebarStore` extension** in `client/lib/stores.ts`:

```typescript
interface SidebarStore {
  // ... existing fields ...
  expandedSpaceIds: Set<string>;
  toggleSpaceExpanded: (id: string) => void;
  collapseAllSpaces: () => void;
}
```

`expandedSpaceIds` is NOT persisted (no `persist` middleware for this key). Resets to empty on app restart. Rationale: sub-space expand state is session-level UI preference, not a persistent setting.
