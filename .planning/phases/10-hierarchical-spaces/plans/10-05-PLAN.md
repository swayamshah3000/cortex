<objective>
Extend `SpaceManager::recluster` so that after the top-level clustering + labeling pass completes, it iterates each top-level Space with > SUB_SPACE_THRESHOLD (50) documents, runs `subspace_detector::detect`, labels the resulting sub-clusters via `llm_labeler::label_sub_cluster`, persists them to the SpaceLabelCache (with parent_id + depth), and pushes them into the flat SpaceManager storage. Top-level Space structs receive their `sub_space_ids` populated at the end of the pass. When a parent's membership Jaccard shift > 20% (existing D-06 gate reused), the parent's ALL existing sub-space cache entries are dropped and recomputed (D-08).

Purpose: Orchestration. All the primitives exist — this plan glues them together in the correct order and with the right cache semantics.

Output:
- `SpaceManager::recluster` extended with sub-space pass. All in-flight in the same function — no separate public API (Plan 06 will handle hyperbolic index rebuild in its own module).
- New private helper `run_sub_space_pass(...)` (or inlined block with clear section comment) that iterates parents, calls detect + build_misc_space + label_sub_cluster, updates cache + flat store.
- Progress events for sub-space labeling reuse the existing `space-labeling-progress` event with sub-space `space_id` values (per UI-SPEC §7 — the SpaceLabelingIndicator copy variant is a frontend concern, not backend).
- 3 unit tests covering the plan_labeling logic for sub-spaces (skip / bootstrap / relabel gate).

Per user decisions:
- D-06: sub-spaces cached in same space_labels.json, keyed by space_id, using the extended SpaceLabelEntry from Plan 01.
- D-08: parent Jaccard > 20% drops ALL its sub-space cache entries + recomputes. Simple + robust.
- D-04: Misc sub-space name is exactly `"Misc"` (case-sensitive) — the frontend depends on this string for dashed-border rendering.
- D-05: sub-space labeling uses `llm_labeler::label_sub_cluster` from Plan 04.
- Discretion: manager retains the sub-space pass inside recluster (rather than a separate public method) — recluster is atomic w.r.t. AppState mutex holders, so keeping the pass inline avoids a second lock cycle.
</objective>
