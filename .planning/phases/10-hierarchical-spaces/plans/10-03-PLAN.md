<objective>
Create the `spaces/subspace_detector.rs` module that runs sub-clustering on parent Spaces exceeding SUB_SPACE_THRESHOLD (50 docs). Wraps existing `clustering::cluster_documents` recursively with k = sqrt(n/2).max(2), rolls up sub-clusters below MIN_SUB_CLUSTER_SIZE (3 docs) into a synthetic "Misc" bucket per parent (HSPC-03).

Purpose: Pure computation module the SpaceManager recluster path (Plan 05) will call once per top-level Space. No I/O, no LLM calls, no async — trivial to unit test.

Output:
- `src-tauri/src/spaces/subspace_detector.rs` new file
- `spaces/mod.rs` extended with `pub mod subspace_detector;`
- 5 unit tests covering threshold gate, misc rollup, empty-misc guard, k formula, and orphan bookkeeping.

Per user decisions:
- D-01: threshold = 50, expressed as `pub const SUB_SPACE_THRESHOLD: usize = 50`
- D-02: recursive k-means, k = sqrt(n/2).max(2). No HDBSCAN (correctly ruled out in RESEARCH.md — parent already trimmed → insufficient density).
- D-04: MIN_SUB_CLUSTER_SIZE = 3. Below that → roll up to Misc. Never drop docs (HSPC-03).
- Naming: "Misc" (exact string, case-sensitive) is the sentinel the frontend checks in SubSpaceCard for the dashed-border variant per UI-SPEC §5.
</objective>
