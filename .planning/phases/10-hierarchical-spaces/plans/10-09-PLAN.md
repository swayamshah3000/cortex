    ### Verification Steps

    **HSPC-01 — Auto-split at 50 docs (10 min)**
    - Navigate to `/spaces`. Identify the largest Space (by document count).
    - Case A: it has > 50 docs → click into `/spaces/:id`. Verify a "Sub-Spaces" section appears with at least 2 sub-space cards.
    - Case B: no Space in the corpus has > 50 docs → SKIP HSPC-01 with a note (unable to exercise threshold gate); user acknowledges.
    - Additionally: navigate to a Space you know has < 50 docs. Verify NO "Sub-Spaces" section renders.

    **HSPC-02 — Sub-space labels + navigation + breadcrumb (10 min)**
    - From the SpaceDetailPage of a large parent, verify sub-space cards show LLM-generated 2-4 word labels (not "Cluster 0", not "Space N — Related", not "Sub-space 1"). Confirm the labels are semantically distinct from the parent's label (D-05 constraint).
    - Click a sub-space card. Verify:
      - URL changes to `/spaces/:sub-id`.
      - The shadcn Breadcrumb shows `Spaces / {parent_name} / {sub_name}`.
      - The parent name in the breadcrumb is a link back to the parent.
      - The ParentContextBanner ("Sub-space of {parent}") renders above the header.
      - Clicking the ParentContextBanner's parent link navigates back to the parent.

    **HSPC-03 — Misc sub-space visual + orphan bookkeeping (10 min)**
    - If any parent Space has ≥ 3 orphan documents (i.e., docs that fell below MIN_SUB_CLUSTER_SIZE), verify a "Misc" sub-space card renders LAST in the grid with a DASHED left border (per UI-SPEC §5).
    - Click into the Misc sub-space. Verify:
      - Its breadcrumb is `Spaces / {parent} / Misc`.
      - The document count matches the number of orphans.
    - If no Misc sub-space exists in your corpus, skip with note (parent's k-means happened to fit all docs into ≥ 3-doc clusters). This is not a failure — it means orphan branch didn't trigger.
    - Verify NO empty Misc card appears anywhere (pitfall #3 in RESEARCH.md — Misc must be conditional).

    **HSPC-04 — Sidebar top 5 + sub-count + inline expand (10 min)**
    - In the sidebar Spaces section, verify EXACTLY 5 top-level Space entries appear (not 6).
    - Verify no sub-space appears as a top-level sidebar entry (pitfall #2 — sub-spaces should render UNDER their parent via expand, never as top-level).
    - Hover a top-level Space that has sub-spaces. Verify a chevron appears (opacity-0 → 100 on hover).
    - Click the chevron. Verify:
      - Chevron rotates 90° via CSS transition.
      - Sub-space list renders indented beneath (pl-8) with smaller 6px dots.
      - URL does NOT change (chevron toggles expand, does not navigate).
    - Click the space name (not chevron). Verify URL changes to `/spaces/:id`.
    - Click the chevron again. Verify sub-list collapses.
    - Collapse the whole sidebar (`Cmd+\` or the toggle at the bottom). Verify chevrons + sub-counts + sub-lists all HIDE — only color dots remain.
    - Restart the app. Verify expanded state RESETS (sub-lists start collapsed) — session-only per D-13.
